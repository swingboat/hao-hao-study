> 本文件为学生端独立 PRD，与 `Operator_Console_MVP_PRD.md` 并列，共同实现主 PRD `SmartLearningAssistant_PRD.md` v1.0 的 §10 v0.1 MVP 范围。
>
> | 版本 | 日期 | 说明 |
> |---|---|---|
> | v0.1 | 2026-06-03 | 首次发布；范围对齐主 PRD §10 v0.1 + 决议 S1/S2/C1/C3 |

## §1 一句话定位

> **学生端是一个 AI 驱动的高考备考 Web：高中学生登录后只需点"开始今日学习"，系统自动凑齐 8–15 题（错题重做 + 间隔复习 + 新题三池合并），25 分钟番茄钟内完成 → 提交 → 看解析与 mastery 变化；用学生最少的判断成本，换最大化的学习路径效率。**

| 维度 | 定义 |
|---|---|
| 核心解决目标 | 替代"题海战术 + 被动补课"，让学生**不做选择**就能走在最短的提分路径上 |
| 核心用户 | 高中学生（高一至高三），目标考试为新课标高考；首批为内部亲友（< 50 人） |
| 核心价值主张 | 学生只做"做题 / 看解析 / 看进度"三件事，所有"今天学什么、什么时候复习"的决策由 AI 推荐器与 Layer 3 间隔复习引擎承担 |
| 部署形态 | Web（移动浏览器友好），与运营端同域不同路径（如 `/` 学生端、`/admin` 运营端） |

## §2 原子化 MVP 功能列表（TAV 格式）

> 共 **17 个原子功能**，分 6 组。每条独立可测，LLM 拿到任意一条即可开工。

### G1 组：入驻与监护人同意

**G1.1 首次访问着陆页**
- **当** 未登录访客访问 `/` **时**，
- **系统执行** 展示产品一句话定位 + "我有账号，登录" + "我没有账号，联系运营开户"两个 CTA，
- **界面显示** 现代审美的极简首页，无任何注册入口（v0.1 仅运营开户，不开放自助注册）。

**G1.2 学生登录**
- **当** 学生在 `/login` 输入运营开户时下发的账号 + 一次性密码 **时**，
- **系统执行** 校验凭据，首次登录强制跳 G1.3 监护人同意页，非首次跳 G2.1 首页，
- **界面显示** 居中卡片含账号 / 密码 / 登录；错误时红字"账号或密码错误"。

**G1.3 监护人同意确认**
- **当** 学生首次登录尚未确认监护人同意 **时**，
- **系统执行** 展示监护人同意书全文 + 必勾"我已与监护人共同阅读并同意" + "我同意数据不用于训练 AI 且可随时导出/删除"两个独立勾选框；提交时写 `student.parent_consent_at = now()` 并写一条 `audit_log`（action=`consent_confirmed`），
- **界面显示** 不勾完两个框"继续"按钮置灰；提交成功跳转首页。

> 监护人同意凭据已由运营在开户时（运营端 F5.2）线下收集；本步骤为学生端二次确认与时间戳留痕，**未确认前禁止访问任何学习功能**。

### G2 组：今日学习首页

**G2.1 首页主视图**
- **当** 已登录已同意学生访问 `/` **时**，
- **系统执行** 计算并返回：今日推荐题数（来自 G3.1 三池合并）、本周完成 Session 数、按章节聚合的 Mastery 4 档环形图（`unlocked_kp_ids` 范围内的 KP），
- **界面显示** 顶部"开始今日学习"主按钮（高亮）+ 当前番茄钟状态 + Mastery 环形图 + "查看错题本"次级入口。

**G2.2 Mastery 4 档可视化**
- **当** 首页加载完成 **时**，
- **系统执行** 对每个 `unlocked_kp_ids` 中的 KP 读取 `knowledge_point_mastery.mastery_score`，按主 PRD §10.2 的 4 档映射：[0, 0.2)=未开始 / [0.2, 0.5)=需要加强 / [0.5, 0.85)=学习中 / [0.85, 1]=已掌握，
- **界面显示** 按章节聚合的环形或条形图，鼠标/手指悬停显示 KP 名 + 具体档位文案；mastery=0 且**曾达到过 ≥ 0.5 的 KP** 文案显示"需要加强（曾掌握后回落）"，从未掌握的显示"需要加强"（决议 Q1=c）。

**G2.3 已锁 KP 的隐藏**
- **当** 渲染 Mastery 可视化或任何 KP 列表 **时**，
- **系统执行** 仅显示 `kp_id ∈ student.unlocked_kp_ids` 的 KP，已锁 KP 完全不在学生端任何视图出现（决议 S2-A），
- **界面显示** 学生看不到任何"灰色锁状"占位，仿佛已锁 KP 不存在。

---

### G3 组：Session 答题闭环（核心）

**G3.1 开始今日学习 / 凑题**
- **当** 学生点击"开始今日学习"且当前无未结束 Session **时**，
- **系统执行** 创建 `learning_session`（status=`in_progress`），按下面 3 池合并去重凑齐 8–15 题：
  - 池 A 错题重做：`mistake_book_entry.status='open' AND student_id=?`
  - 池 B 间隔复习：`spaced_review.next_review_at <= now() AND student_id=?`
  - 池 C 新题：`mastery_score < 0.5 AND kp_id ∈ unlocked_kp_ids` 的 KP 内随机取题
  - 三池统一加过滤 `kp_id ∈ unlocked_kp_ids`（决议 S2-A），同 KP 多池命中**合并去重**只取 1 题（决议 S2-衍生 2）
- **Planner 设计** 详见 `docs/Student_Question_Planner_v0.1.md`。第一版固定 `daily_mixed / chapter_focus / mistake_focus` 三种模式，AI 只在单个题位内生成题目，不参与全局调度。
- **界面显示** 进入答题页，顶部 25 分钟番茄钟开始倒计时 + 题目进度"1 / 12"。

**G3.2 单题作答**
- **当** 学生在答题页选择选项（choice）或填入答案（fill_in）**时**，
- **系统执行** 暂存答案到客户端（不立即写库），允许学生在提交前自由前后翻题修改，
- **界面显示** 题干 + 选项/填空框 + "上一题 / 下一题 / 提交全部"三按钮；已答题在题号导航条标蓝点。

**G3.3 一次性提交**
- **当** 学生点击"提交全部"或番茄钟到 0 自动提交 **时**，
- **系统执行** 在单一事务内完成：
  1. 批量写入 `question_attempt`（每题 1 行，含 `is_correct`）
  2. 按主 PRD §10.2 增减规则更新 `knowledge_point_mastery.mastery_score`（clamp [0, 1]）
  3. 答错的题：`mistake_book_entry` upsert（error_count += 1，status=`open`）；连续 2 次答对该题则 status=`resolved`（决议 S2-N=2）
  4. 答错且根因非 computational 的题：`spaced_review` upsert（idx=0，next_review_at = now() + 1 day，触发器 A）
  5. 新学首次 attempt 的 KP：`spaced_review` upsert（idx=0，触发器 B）
  6. 关闭 `learning_session`（status=`completed`，ended_at=now()）
- **界面显示** 转圈"批改中"，完成后跳转到 G3.4 解析页。

**G3.4 解析与 Mastery 变化展示**
- **当** Session 提交成功 **时**，
- **系统执行** 渲染本次 Session 全部题目的"题干 / 你的答案 / 正确答案 / 对错 / 解析（solution_text）"，并展示 mastery 变化摘要（哪些 KP 涨了、涨了多少、是否升档），
- **界面显示** 顶部成绩条"答对 9 / 12（75%）+ 用时 22 分钟"，下方逐题展开卡片，错题红边、对题绿边；底部固定"返回首页"按钮。

**G3.5 中断与恢复**
- **当** 学生在 Session 进行中关闭浏览器或刷新 **时**，
- **系统执行** 不在服务端保存中间答案；下次进入首页若存在未完成 Session（`status='in_progress'` 且超过 25 分钟），自动标记为 `abandoned` 并允许开新 Session，
- **界面显示** 首页若检测到上一次 abandoned Session，提示"上次学习未完成，已自动结束"，主按钮恢复为"开始今日学习"。

> v0.1 不做"答题进度本地保存 / 跨设备恢复"，刷新即丢答案——这是有意为之的复杂度切割。

### G4 组：错题本

**G4.1 错题本列表**
- **当** 学生进入 `/errors` **时**，
- **系统执行** 读取 `mistake_book_entry.status='open' AND student_id=?` 的全部记录，按 KP 分组（KP 名作为分组标题），同 KP 内按 `error_count` 倒序，
- **界面显示** 折叠列表，每组标题显示"KP 名 + 累计 N 道未掌握错题"；展开后显示每条错题的题干前 60 字 + "重做"按钮。

**G4.2 错题重做入口**
- **当** 学生点击错题列表中某条的"重做"**时**，
- **系统执行** 单题模式打开（不进 Session 流程），提交后立即批改并按 G3.3 的步骤 3 更新 mistake_book_entry（连续 2 次对则 resolve），
- **界面显示** 全屏单题作答页 + 提交后立即显示对错与解析；底部"再来一题"（同 KP 内换一题）/ "返回错题本"。

**G4.3 错题自动消解提示**
- **当** 某条 mistake_book_entry 因连续 2 次对从 open 转为 resolved **时**，
- **系统执行** 在 G3.4 解析页或 G4.2 重做完成页弹一次性提示，
- **界面显示** 绿色 toast："恭喜！『XX 知识点』的这道题已掌握，从错题本移除。"

---

### G5 组：个人中心与合规

**G5.1 个人中心**
- **当** 学生进入 `/me` **时**，
- **系统执行** 读取 `student` 基本信息 + 累计学习时长（来自 learning_session 时长求和）+ 累计答题数（question_attempt 计数）+ 累计错题已 resolve 数，
- **界面显示** 卡片化展示：姓名 / 年级 / 目标考试 / 加入日期 / 学习时长 / 答题数 / 错题攻克数；下方"导出我的数据"+ "注销账户"两按钮。

**G5.2 数据导出**
- **当** 学生点击"导出我的数据"**时**，
- **系统执行** 异步打包该学生的 student / question_attempt / knowledge_point_mastery / mistake_book_entry / spaced_review / learning_session 6 个 JSON 文件为 ZIP，写 `audit_log`（action=`export_data_self`），
- **界面显示** 提示"数据准备中，约 30 秒后自动下载"；完成后浏览器自动下载文件 + toast"导出完成"。

**G5.3 账户注销自助**
- **当** 学生点击"注销账户"且通过二次确认 **时**，
- **系统执行** 弹出红色确认对话框（输入"注销我的账户"7 字才可继续），通过后软删 student + 硬删另 5 张表，写 `audit_log`（action=`delete_self`），清除 cookie 跳登录页，
- **界面显示** 红色二次确认对话框 + 灰色不可恢复警告；完成后跳到一个静态"账户已注销"提示页。

---

### G6 组：通用与异常

**G6.1 顶部导航与登出**
- **当** 学生处于任何已登录页面 **时**，
- **系统执行** 顶部导航始终显示"今日学习 / 错题本 / 我的"三入口 + 右上角学生姓名（点击下拉含"登出"），
- **界面显示** 移动端折叠为汉堡菜单；登出清除 cookie 跳登录页。

**G6.2 Session 重复开启拦截**
- **当** 学生在已有 in_progress Session 时再次点击"开始今日学习"**时**，
- **系统执行** 直接路由到现有 Session 答题页（不创建新 Session），
- **界面显示** 无额外提示，无缝继续。

**G6.3 空数据态**
- **当** 学生的 `unlocked_kp_ids` 为空，或三池凑题结果为 0 **时**，
- **系统执行** 不创建 Session，
- **界面显示** 首页主按钮置灰，下方文案"今天没有需要学习的内容，请联系老师扩展学习范围。"；Mastery 环形图位置展示空状态插画。

## §3 技术排他边界（Out of Scope）

> 以下功能 **v0.1 绝对不构建**。LLM 实现时遇到这些需求，应直接拒绝或留 TODO，不得自行扩展。

### §3.1 题型与输入
- ❌ 拍照上传题目 / OCR（对应主 PRD ⏸ B4）。
- ❌ 解答题（essay）作答与判分（决议 Q2=a）。
- ❌ 讲述题 / 费曼题型（v0.5 再做）。
- ❌ 综合题 / 跨 KP 大题（v1.5 再做）。
- ❌ 学生端任何形式的题目上传 / 修改 / 反馈打分。

### §3.2 学习路径自主权
- ❌ 学生自选 KP 学习。
- ❌ 学生自选题做。
- ❌ 学生跳过推荐题 / 重新抽题。
- ❌ 学生在 Session 内换题。
- ❌ 学生自定义每日学习时长 / 题数。
- ❌ 学生选择推荐器配比（`recommendation_mix_override` 决议 C3-D3 仅运营可改）。

### §3.3 推荐器与复习
- ❌ 7 池完整推荐器（v0.1 仅 3 池：错题 / Layer 3 / 新题）。
- ❌ Layer 3 触发器 C（讲述达标）/ D（长间隔保鲜）—— 仅做 A（答错）/ B（新学）。
- ❌ 临考期（≤ 30 / ≤ 7 天）模式切换与间隔压缩（对应主 PRD ⏸ B2）。
- ❌ 灰度池 / 挑战题池。
- ❌ 阶段化配比（sprint / consolidation / exploration）。

### §3.4 错题本能力
- ❌ 错题根因分类显示（conceptual / methodological / comprehension / time_pressure 等）—— v0.1 不分类。
- ❌ 错题变种召回（同 KP 不同题号自动串）。
- ❌ 错题两阶段流转（`open_pending_material → open`，决议 D1=A 延后到 v0.2）。
- ❌ 错题手动标记掌握 / 手动 resolve。
- ❌ 错题归档（resolve 即永久 resolve，不可 reopen，对应决议 D2=a）。

### §3.5 个性化与画像
- ❌ 学生学习画像 / 抗拒识别（对应主 PRD ⏸ B5）。
- ❌ AI 对话 / 一对一辅导聊天框。
- ❌ 学习目标自定义（学生看到的 target_score / mastery_threshold 来自运营 GoalInstance，不可改）。
- ❌ 自定义学习提醒推送 / 邮件通知。

### §3.6 社交与外部
- ❌ 学生间排行榜 / 互动 / 评论。
- ❌ 监护人/家长端登录与查看（v0.1 监护人同意仅留痕，不开账号）。
- ❌ 教师端 / 班级群组。
- ❌ 第三方账号登录（OAuth）。
- ❌ 分享学习成果到社交平台。

### §3.7 内容与体验
- ❌ 章节进度自动同步 / 超前解锁（对应主 PRD ⏸ B6）。
- ❌ 多教材版本切换（对应主 PRD ⏳ B7 字段缺失）。
- ❌ 多学科支持（v0.1 仅 1 个学科，由运营硬塞 subject）。
- ❌ 暗色模式。
- ❌ 多语言 i18n（v0.1 仅简体中文）。
- ❌ 移动原生 App（v0.1 仅 Web，移动浏览器友好即可）。
- ❌ 离线模式 / PWA 缓存。

### §3.8 数据与安全
- ❌ 学生自助修改密码（v0.1 联系运营重置）。
- ❌ 二次密码 / 双因素认证。
- ❌ 学习日志可视化时间线 / 详细 question_attempt 历史浏览（隐私简化，仅在导出 ZIP 中提供）。
- ❌ 部分数据导出 / 时间范围筛选导出。
- ❌ 撤销注销 / 数据找回。

## §4 编号用户旅程（User Flow）

### §4.1 主旅程 V1 — 首次入驻到完成首个 Session（最小价值闭环）

> 总耗时目标：**学生 ≤ 35 分钟**完成。

1. 学生收到运营线下下发的账号 + 一次性密码，访问首页（G1.1），点击"我有账号，登录"。
2. 在登录页输入凭据（G1.2），系统识别为首次登录，强制跳监护人同意页（G1.3）。
3. 学生与监护人共同阅读同意书，勾选两个独立勾选框，提交后写 `parent_consent_at`，跳首页。
4. 首页展示 Mastery 环形图（G2.1 / G2.2，全部显示"未开始"）+ "开始今日学习"主按钮。
5. 学生点击"开始今日学习"（G3.1），系统按三池合并凑 8–15 题，进入答题页。
6. 学生在 25 分钟番茄钟内逐题作答（G3.2），可前后翻题修改，最后点击"提交全部"（G3.3）。
7. 系统在事务内更新 question_attempt / knowledge_point_mastery / mistake_book_entry / spaced_review，跳解析页（G3.4）。
8. 学生查看每题对错与解析、查看 mastery 涨幅，点"返回首页"。**首次价值闭环达成**——首页 Mastery 图首次出现非"未开始"档位的 KP。

### §4.2 日常旅程 V2 — 每日打开做一次 Session

1. 学生登录（G1.2，非首次直接进首页）。
2. 首页主按钮"开始今日学习"，点击进入 Session（G3.1）。
3. 答题 + 提交（G3.2 / G3.3）。
4. 看解析（G3.4）+ 看 Mastery 变化 + 看是否有错题被自动 resolve（G4.3）。
5. 返回首页，看到本周 Session 数 +1 + Mastery 环形图更新。

### §4.3 错题修复旅程 V3 — 主动攻克错题

1. 学生从首页点"查看错题本"进入 G4.1。
2. 选择某 KP 下的某条错题，点"重做"进入 G4.2 单题模式。
3. 提交后立即看解析 + mistake_book_entry.error_count 更新；若已是该题第 2 次答对，弹 G4.3 提示该题已掌握。
4. 点"再来一题"继续同 KP 错题，或"返回错题本"。

### §4.4 合规旅程 V4 — 数据导出 / 注销自助

1. 学生进入"我的"页面（G5.1）。
2. 点"导出我的数据"（G5.2），等待 30 秒后浏览器自动下载 ZIP。
3. 或点"注销账户"（G5.3），输入"注销我的账户"7 字确认，账户即刻不可访问。

## §5 成功标准（可被自动化测试验证）

> 所有指标分两类：**功能正确性**（必须 100% 通过，CI 拦截）与 **质量水位**（验收阈值，灰度后达标）。

## §6 数据模型摘要（学生端只读 + 写入边界）

> 完整字段表见主 PRD §3 / §10.4。本节仅列出学生端的读/写权限边界。

| 表 | 学生端读 | 学生端写 | 触发场景 |
|---|---|---|---|
| `student` | ✅ 自己一行 | ✅ 仅 `parent_consent_at`（G1.3） | 监护人同意 |
| `subject` | ✅ 全表只读 | ❌ | KP 学科展示 |
| `knowledge_point` | ✅ 仅 `id ∈ unlocked_kp_ids` | ❌ | Mastery 图 / 错题本分组 |
| `question` | ✅ 仅 Session/错题本中分配的题 | ❌ | 答题页 / 解析页 |
| `learning_session` | ✅ 自己 | ✅ 创建 / 完成 / abandoned 状态机 | G3.1 / G3.3 |
| `question_attempt` | ✅ 自己 | ✅ 仅通过 G3.3 批量插入 | Session 提交 |
| `knowledge_point_mastery` | ✅ 自己 + 仅 `unlocked_kp_ids` | ✅ 仅通过 G3.3 / G4.2 间接更新 | 答题后派生 |
| `mistake_book_entry` | ✅ 自己 | ✅ 仅通过 G3.3 / G4.2 间接更新 | 答题后派生 |
| `spaced_review` | ❌ 学生不见 | ✅ 仅通过 G3.3 间接更新 | 答题后派生 |
| `audit_log` | ❌ | ✅ 仅 action ∈ {`consent_confirmed`, `export_data_self`, `delete_self`} 由系统写 | 合规事件 |

强约束：

- 学生端**任何接口**返回的数据必须做 `kp_id ∈ unlocked_kp_ids` 过滤，未做过滤的接口视为安全漏洞。
- 学生端**禁止**直接修改 `student.unlocked_kp_ids`、`knowledge_point_mastery.mastery_score`、`mistake_book_entry.status` —— 这些字段仅由后端服务通过 G3.3 / G4.2 的事务更新。
- `parent_consent_at` 一旦写入不可修改不可清空（除注销时随 student 软删带走）。
