# A1 三天学习闭环验收清单

> 本清单用于验证“高二学生一轮复习 MVP”是否真的能形成每日学习闭环。
>
> 验收对象：seed 学生 `niki / Niki / 高二 / 高考 2027`。
>
> 验收目标：学生连续 3 天使用后，能清楚知道今天练什么、为什么练、哪里变强、明天还会补什么。

## 前置条件

- `main`、`feat/web`、`feat/admin` 已同步到同一基线。
- 数据库已迁移并 seed。
- `niki` 学生存在，已完成监护人同意。
- `niki.unlocked_kp_ids` 至少包含 10 个高二数学一轮复习知识点。
- 每个已解锁知识点至少有 3 道可作答题目；建议目标为 5 道。
- web端不得展示数据库主键、provider id、job id、pool 原值、fallback、内部路径。

## 第 1 天：首次形成学习记录

### 目标

验证学生能从首页开始一组今日复习，提交后看到解析，并在首页或进度页看到知识点状态变化。

### 操作步骤

1. 使用 `niki` 登录web端。
2. 进入首页。
3. 记录首页展示的今日题量、预计用时、推荐内容说明。
4. 点击“开始今日复习”或当前主按钮。
5. 完成整组题目作答。
6. 提交全部答案。
7. 查看结果页。
8. 返回首页和学习进度页。

### 验收标准

- [ ] 首页主按钮明确，学生能理解下一步动作。
- [ ] 首页说明今天练什么，不能只显示抽象数据。
- [ ] 答题页题干、选项、公式、题图显示正常。
- [ ] 提交后进入结果页。
- [ ] 结果页展示“你的答案 / 正确答案 / 解析 / 对错”。
- [ ] 至少一个知识点从“未开始”进入“学习中”或“需要加强”。
- [ ] 页面没有出现 `kp_id`、`question_id`、`pool`、`ai_generated`、`fallback` 等内部标识。

## 第 2 天：错题与间隔复习召回

### 目标

验证系统能把第 1 天的错题或到期复习内容重新安排给学生，而不是每天只做新题。

### 操作步骤

1. 将系统时间或测试数据调整到第 2 天可复习状态。
2. 使用 `niki` 进入首页。
3. 查看首页今日推荐说明。
4. 开始今日复习。
5. 检查本组题是否包含错题复习或间隔复习来源。
6. 提交后查看结果页。

### 验收标准

- [ ] 今日推荐说明能表达“回炉错题”或“安排到期复习”。
- [ ] 今日题组中至少包含一类复习内容：历史错题或到期复习知识点。
- [ ] 如果复习题做对，结果页能体现对应知识点有改善。
- [ ] 如果复习题继续做错，错题仍保留在后续复习路径中。
- [ ] web端不展示 `spaced_review`、`mistake_variant`、`error_review` 等工程词。

## 第 3 天：一轮复习进度感

### 目标

验证学生连续使用后，能从首页和进度页看到“我在一轮复习中推进了哪里”。

### 操作步骤

1. 使用 `niki` 进入首页。
2. 查看本周完成次数。
3. 查看今日复习任务说明。
4. 完成今日复习并提交。
5. 进入学习进度页。
6. 按章节查看已掌握、学习中、需要加强的知识点。

### 验收标准

- [ ] 首页能展示本周完成情况。
- [ ] 进度页按章节组织知识点。
- [ ] 学生能快速找到“需要加强”的知识点。
- [ ] 学生能看到哪些知识点已经掌握或正在学习。
- [ ] 结果页和进度页之间的变化一致。

## 通过标准

本轮 A1 通过必须同时满足：

- [ ] 三天主旅程都能完成，没有阻断性错误。
- [ ] 学生每天都能理解“今天为什么练这些”。
- [ ] 至少一次出现错题或间隔复习召回。
- [ ] 至少一次出现知识点状态变化。
- [ ] web端无内部工程标识暴露。

## 失败处理

如果验收失败，按以下优先级处理：

1. 阻断学生完成练习的问题先修。
2. 内部字段或工程状态暴露的问题立即修。
3. 学生不理解“为什么练这些”的问题进入首页任务台优化。
4. 题库不足导致无法连续 3 天练习的问题进入题库水位任务。
