# Goal Template 设计文档

> 本文档基于"AI 智能助学工具"产品讨论沉淀，专注于讲清楚 **Goal Template（目标模板）** 这一核心运营资产的设计原理、构成、构建方法。
>
> 适用范围：MVP 阶段 —— 高考新课标 I 卷数学（覆盖安徽、山东、广东、江苏、河北、湖北、湖南、福建、浙江、辽宁等省份）。
>
> 文档版本：v0.1（讨论稿）

---

## 1. 为什么需要 Goal Template（Why）

### 1.1 产品里"目标"的本质

学生在产品里设定的"目标"（如"2027 年高考"）不是一个静态字符串，而是一个由**考试要求 + 学生起点 + 截止时间 + 每日预算**共同决定的复杂结构。其中前半部分（考试要求）对所有报考同一场考试的学生都是一样的，应该作为**可复用的运营资产**沉淀下来；后半部分（个人起点、截止时间、每日预算）才是学生个人数据。

由此引出两个分离的概念：

- **Goal Template（目标模板）**：运营定义的考试要求底座。"2027 年高考新课标 I 卷数学"是一个模板。
- **Goal Instance（目标实例）**：学生在某个模板上叠加自己个人数据后形成的实际目标。"张三的 2027 年高考目标"是一个实例。

**这种分离的必要性**：

1. 千万学生共享同一份高质量考试要求数据，避免重复建设。
2. 模板更新（如考纲调整）时，可以通过版本管理批量影响学生。
3. 学生数据和教研资产解耦，便于权限管理和合规审计。

### 1.2 没有 Goal Template 的后果

如果系统不维护这层抽象，会出现以下问题：

- **推荐失准**：AI 不知道"考点权重""能力维度"，只能盲推题目。
- **诊断粗糙**：学生错题后只能说"这道题错了"，无法定位到知识点 / 能力点。
- **无权威背书**：不能向家长说明"我们对标教育部官方框架"，权威性立不住。
- **教研资产无法沉淀**：每次新增学科都要从零开始，无法形成壁垒。

### 1.3 Goal Template 解决的核心问题

| 问题 | Goal Template 的回答 |
|---|---|
| 学生该学什么？ | 模板的"知识点 + 课标条目"清单 |
| 学到什么程度算会？ | 模板的"能力点 + 学业质量水平" |
| 考试会考什么？ | 模板的"考点 + 权重" |
| 卷子怎么构成？ | 模板的"题型 / 分值 / 时长结构" |
| AI 该推什么题？ | 模板的"考点权重 + 难度分布" |
| 怎么向家长解释？ | 模板的"课标锚点 + 评价体系映射" |

## 2. Goal Template 包含哪些信息（What）

### 2.1 整体结构

一份完整的 Goal Template 由 **4 个数据源**支撑、**6 大组件**构成：

```
Goal Template
├── 元信息
├── 内容侧组件（来自《课程标准》）
│   ├── 课标条目库
│   ├── 知识点 DAG
│   └── 核心素养体系
├── 能力侧组件（来自《中国高考评价体系》）
│   ├── 四层维度
│   └── 四翼维度
├── 卷子结构组件（来自省考试院 + 真题）
│   ├── 总分 / 时长
│   ├── 题型分布
│   └── 难度分布
├── 考点体系组件（教研定义 + 真题反推）
│   ├── 考点表
│   └── 考点-知识点关联
├── 权重组件（真题统计）
│   ├── 考点权重
│   └── 题型 / 难度权重
└── 真题库
    └── 题目 ↔ 考点 的标注关系
```

### 2.2 四类数据源（解决信息从哪来的问题）

| 数据源 | 回答什么问题 | 颁布方 | 官方性 |
|---|---|---|---|
| 《课程标准》 | 教什么、学到什么水平 | 教育部 | 是 |
| 《中国高考评价体系》 | 怎么考、考什么能力 | 教育部考试中心 | 是 |
| 省级考试说明 / 实施方案 | 卷子怎么构成 | 各省教育考试院 | 是 |
| 历年真题 + 命题趋势分析 | 实际怎么考、怎么变 | 真题官方 / 分析非官方 | 真题官方 |

**重要澄清**：
- 《课程标准》是全国统一文件，不分省份不分卷种。
- "新课标 I 卷"是试卷，不是课标。卷子按课标命题，但不等于课标。
- 2020 年后教育部已停发《普通高中考试大纲》，课标和高考评价体系共同承担了考纲的角色。

### 2.4 内容侧组件

#### 2.4.1 课标条目库

来源：《普通高中数学课程标准（2017 年版 2020 年修订）》PDF 抽取。

| 字段 | 示例 |
|---|---|
| 条目 ID | std_math_4_2_1 |
| 学科 | 数学 |
| 课标版本 | 2017 年版 2020 修订 |
| 主题 | 函数概念与性质 |
| 单元 | 一次函数 |
| 原文 | "理解函数的概念，了解构成函数的要素..." |
| 关联核心素养 | 数学抽象、数学运算 |

**预估条目数（数学）**：200-300 条。

#### 2.4.2 知识点 DAG

DAG = Directed Acyclic Graph（有向无环图）。**不能用树结构**，因为一个知识点常常有多个前置（如"一元二次方程求根公式"依赖"平方根""配方法""代数式运算"等多个前置）。

**为什么是 DAG**：
- 一个知识点可以有**多个**父节点（多个前置）
- 顺着箭头永远回不到起点（无环 —— 否则知识点切分错了）
- 教材章节只是 DAG 的一种线性遍历，不能反过来当主结构

**用途**：
- **诊断溯源**：学生在 A 上错，沿 DAG 反向查 A 的前置，定位真正根因
- **学习路径生成**：要学 B，沿 DAG 反推所有未掌握前置，按拓扑顺序排计划
- **阻塞检测**：A 没过，所有以 A 为前置的下游知识点暂时不推
- **跨教材版本复用**：知识点 DAG 共享，教材只是不同的呈现路径

**数据模型**：

```
知识点表
  - id, 名称, 学科, 学段, 粒度层级
  - 关联课标条目 ID (1:1 或 1:N)
  - 关联核心素养 (可多个)
  - 是否高频考点

知识点-知识点 前置关系表
  - from_id (前置)
  - to_id (后继)
  - 强度 (强依赖 / 弱依赖)
  - 依赖说明 (可选，便于人审)
```

**预估规模（数学）**：
- 知识点 200-400 个（单学科超过 400 通常说明切太细）
- 前置关系边 400-800 条

**粒度准则**：一个知识点对应的最小学习时间约 15-30 分钟。再细就拆成"知识点 + 子能力"两层。

#### 2.4.3 核心素养体系

直接抄《课程标准》，不要自己造：

| 素养（数学） | 课标定义水平 |
|---|---|
| 数学抽象 | 水平 1 / 2 / 3 |
| 逻辑推理 | 水平 1 / 2 / 3 |
| 数学建模 | 水平 1 / 2 / 3 |
| 直观想象 | 水平 1 / 2 / 3 |
| 数学运算 | 水平 1 / 2 / 3 |
| 数据分析 | 水平 1 / 2 / 3 |

合计 18 个能力点。高考主要锚定水平 2-3。

### 2.5 能力侧组件（高考评价体系）

来源：《中国高考评价体系》及其《说明》（教育部考试中心，2019 年发布，2020 年起取代《考试大纲》）。

**一核四层四翼**框架：

- **一核**（考查目的）：立德树人、服务选才、引导教学
- **四层**（考查内容）：核心价值、学科素养、关键能力、必备知识
- **四翼**（考查要求）：基础性、综合性、应用性、创新性

**对每个考点 / 每道题，需要打两组标签**：

| 维度 | 标签 |
|---|---|
| 四层 | 多选：核心价值 / 学科素养 / 关键能力 / 必备知识 |
| 四翼 | 多选：基础性 / 综合性 / 应用性 / 创新性 |

**与课标核心素养的关系**：四层四翼是命题侧框架，核心素养是教学侧框架。同一道题既挂"核心素养"又挂"四层四翼"，两者不冲突。

### 2.6 卷子结构组件

来源：省级考试院发布的考试实施方案 + 历年真题统计。

**新课标 I 卷数学的典型卷子结构**（以近年高考为准，每年可能微调）：

| 字段 | 取值 |
|---|---|
| 总分 | 150 分 |
| 考试时长 | 120 分钟 |
| 是否分文理 | 否 |
| 单选题 | 8 题 × 5 分 = 40 分 |
| 多选题 | 3 题 × 6 分 = 18 分 |
| 填空题 | 3 题 × 5 分 = 15 分 |
| 解答题 | 5 题，分值通常为 13+15+15+17+17 = 77 分 |
| 计算器 | 不允许 |
| 选做题 | 已取消 |

**用途**：
- 模拟卷生成必须按这个结构出
- 学生练习时显示"本题相当于高考 X 分"
- 计时训练按真实时长比例

### 2.7 考点体系组件

**考点定义**：考点 = 知识点 × 能力要求 × 题型场景 的组合。

**关键认知**：
- 考点**没有官方定义**，是教研行业的工程化术语
- 各家教辅切的考点表都不一样，**切法本身就是产品壁垒**
- 不要对家长宣称"对标官方考点"，应说"对标课标 + 真题归纳"

**考点与知识点的关系**：多对多
- 1 个知识点可衍生多个考点
- 1 个考点可能涉及多个知识点

**数据模型**：

```
考点表
  - id, 名称, 学科
  - 主关联知识点 ID
  - 主关联能力点 ID (核心素养 + 水平)
  - 题型场景描述
  - 四层 / 四翼标签

考点-知识点 关联表 (多对多)
  - 考点 ID, 知识点 ID, 角色 (主考 / 辅考)

题目表
  - id, 题干, 答案, 解析, 难度系数
  - 关联考点 ID (一道题可能挂多个考点)
  - 关联能力点 ID
```

**预估规模（数学）**：200-400 个考点。

### 2.8 权重组件

来源：近 5-10 年真题统计 + 八省联考 / 省质检卷补充样本。

**A. 考点权重**（采样推荐的核心依据）：

```
考点权重表
  - 考点 ID
  - 模板版本
  - 出现频次 (近 5 年)
  - 累计分值
  - 分值占比
  - 难度分布 (易 / 中 / 较难 / 难 各占比)
```

**B. 题型分布权重**：用于模拟卷生成。

**C. 难度分布权重**：用于根据学生水平选合适难度的题。

**展示给学生**：不要直接显示数字，用视觉化方式：
- "本考点近 5 年高考出现 4 次，平均占 8 分"
- 练习总结页：饼图显示本次考点分布

### 2.9 真题库

每道真题需要的标注字段：

```
真题表
  - id, 年份, 卷种, 题号, 题型, 分值
  - 题干, 标准答案, 评分细则 (步骤分拆解)
  - 官方解析 (来自《高考试题分析》)
  - 难度系数 (如有)
  - 关联考点 ID (一对多)
  - 关联知识点 ID (一对多)
  - 关联能力点 ID (核心素养 + 四层四翼)
  - 命题意图 (来自官方解读)
```

**预估规模**：近 5 年真题约 100-200 道，加联考 / 质检约 500-1000 道。

## 3. Goal Template 怎么建立（How）

### 3.1 所需资料清单

**必备资料**：

| 资料 | 来源 | 成本 | 解决的问题 |
|---|---|---|---|
| 《普通高中数学课程标准（2017 年版 2020 年修订）》 | 教育部官网（免费） | 0 | 内容侧底座 |
| 《高中数学课标解读》 | 出版社 | 50-100 元 | 课标条目内涵的"使用说明书" |
| 《中国高考评价体系》 | 人教社 | 30-50 元 | 能力侧框架定义 |
| 《中国高考评价体系说明》 | 人教社 | 30-50 元 | 一核四层四翼**如何用于命题**的实操指南 |
| 教育部考试中心《高考试题分析》数学分册（近 5 年） | 高教社 | 200-300 元 | 真题官方命题解读 + 部分难度系数 |
| 人教版高中数学全套教材 | 人教社 | 已有 | 教学呈现顺序 |
| 人教版数学**教师用书**全套 | 人教社 | 300-500 元 | 重难点 + 知识点前置关系 |
| 安徽省高考实施方案 + 考试说明 | 安徽考试院官网 | 免费 | 卷子结构权威数据 |
| 近 3 年八省联考 + 省质检卷 | 教辅汇编 | 100-200 元 | 扩大权重统计样本 |
| 1-2 本主流考点教辅（《五三》《一本》等） | 书店 | 100-200 元 | 考点切分对照参考 |
| 真题评分细则 / 步骤分拆解 | 省教研室 / 教辅 | 视来源 | 错题分步骤归因 |

**总资料成本**：约 1000-1500 元。

### 3.2 所需"过程性产物"（资料以外的）

**A. 知识点切分 SOP（约 5-10 页）**
- 粒度准则（一个知识点对应 15-30 分钟学习时间）
- 命名规范（统一用语，避免同义重复）
- 拆分判据（什么情况下拆，什么情况下合并）

**B. AI 标注 + 人工审核流水线**
- LLM 抽取课标条目 → 人审
- LLM 给真题打考点 / 知识点 / 能力点标签 → 人审
- LLM 聚类相似题 → 人命名考点 → 人审
- 每一步定义 SLA（单题人审平均时长 + AI 准确率要求）

**C. 版本管理与发布机制**
- 模板用 Git 还是数据库管理
- 改 / 发布的权限
- Goal Instance 怎么绑定模板版本号（**MVP 必须预留版本号字段**）

### 3.3 加工的人（必须明确）

三种现实选项：

| 选项 | 成本 | 质量 | 节奏 |
|---|---|---|---|
| 自己上 | 时间 30-45 人天 | 取决于个人教学经验 | 慢 |
| 兼职高中数学老师 | 1.5-3 万元（80-150 元 / 小时） | 较高 | 较快 |
| 师范生 / 教研在读研究生 | 较低 | 需要更多 review | 中等 |

**MVP 推荐**：兼职老师 + AI 辅助 + 创始人审核。

### 3.4 建设流程（分阶段）

**阶段 1：资料整理与数字化（约 1 周）**
- 所有 PDF 资料 OCR / 解析
- 真题集结构化（题干 / 答案 / 解析分字段）
- 写好知识点切分 SOP

**阶段 2：内容侧组件建设（约 3-4 周）**
- AI 抽取课标条目，人工审核
- 教研老师切分知识点（参照教材 + 教师用书 + 教辅）
- 建立知识点 DAG 前置关系
- 关联核心素养

**阶段 3：考点体系建设（约 3-4 周）**
- AI 对真题做聚类，每个簇作为候选考点
- 教研老师命名 + 审核考点
- 对照教辅查漏补缺
- 建立考点-知识点关联

**阶段 4：真题打标与权重统计（约 3-4 周）**
- AI 给每道真题打考点 / 知识点 / 能力点 / 四层四翼 标签
- 人工审核（特别是能力点和四层四翼）
- 程序统计考点权重
- 补全评分细则

**阶段 5：质量验证与发布（约 1 周）**
- 教研老师随机抽样验证标注准确率
- 跑通"学生 → Goal Instance → 推荐 → 答题 → 诊断"的全链路 demo
- 模板 v1.0 发布

**总周期**：3-4 个月（兼职老师每周 2-3 天投入的节奏）。

### 3.5 AI 在每步能做什么 / 不能独立做什么

| 任务 | AI 能做 | AI 不能独立做 |
|---|---|---|
| 课标条目抽取 | 从 PDF 抽结构化条目 | 判断条目内涵 |
| 知识点切分 | 给出候选切分方案 | 定义粒度（教学决策） |
| 知识点前置关系 | 从教材文本推断候选 | 区分强 / 弱依赖 |
| 考点定义 | 真题聚类做候选 | 命名 + 权威 sign-off |
| 真题打考点标签 | 给出候选标签 | 边界争议题的归属 |
| 真题打能力点标签 | 给出候选标签 | 主观性强，必须人审 |
| 评分细则 | 提取参考答案要点 | 细则的合理性判断 |

**核心原则**：AI 出草稿 → 教研专家审核 → 运营发布 → 持续修订。

## 4. 具体例子：一道数学题如何匹配 Goal Template

> **声明**：以下题目为符合新课标 I 卷命题风格的代表性示例，**非 2025 高考真题逐字复述**。映射方法论与真题无差异。

### 4.1 示例题目

> **题目**：已知数列 {aₙ} 满足 a₁ = 1，aₙ₊₁ = 2aₙ + 1（n ∈ N*）。
>
> （1）求数列 {aₙ} 的通项公式；
>
> （2）设 bₙ = n / (aₙ + 1)，求数列 {bₙ} 的前 n 项和 Sₙ。
>
> （满分约 13 分，属解答题中等难度）

### 4.2 元信息映射

| 字段 | 取值 |
|---|---|
| 所属模板 | 新课标 I 卷数学 2027 (v1.0) |
| 题号 | 解答题第 1 题 |
| 题型 | 解答题 |
| 分值 | 13 分 |
| 占总分比 | 8.7% |
| 难度位 | 中等 |
| 数据来源 | 考试院公告 + 历年真题统计 |

### 4.3 内容侧映射

**触及的知识点（多对多关系的体现）**：

| 知识点 | 角色 | 课标锚点 |
|---|---|---|
| 数列的概念与表示 | 必备 | 选修二 第四章 内容要求 |
| 递推关系 | 必备 | 选修二 第四章 |
| 等比数列的定义与通项公式 | 主考 | 选修二 第四章 |
| 错位相减法 | 主考（第 2 问） | 学业要求"特殊数列求和方法" |
| 数列求和 | 必备 | 选修二 第四章 |
| 构造法（通性通法） | 隐性 | 不在课标，属教研定义 |

**DAG 前置链**（解题所依赖的知识点串）：

```
代数式运算 → 数列概念 → 递推关系
                          ↓
                  构造法（aₙ + 1 视为新数列）
                          ↓
                  等比数列通项 ← 求 aₙ
                                    ↓
                           带入求 bₙ
                                    ↓
              错位相减法 ← 求 Sₙ
```

**诊断价值**：学生做不出第 (1) 问，系统沿 DAG 反向溯源，判断是构造法 / 等比数列通项 / 递推概念 哪个前置出了问题。

### 4.4 能力侧映射

**核心素养标签**（来自课标）：

| 素养 | 水平 | 体现 |
|---|---|---|
| 数学运算 | 水平 2-3 | 整道题运算量大，错位相减易错 |
| 逻辑推理 | 水平 2 | 从递推到构造的推理 |
| 数学抽象 | 水平 2 | 把递推抽象为等比数列 |

**高考评价体系标签**（来自《评价体系》）：

| 维度 | 标签 |
|---|---|
| 四层 | 必备知识 + 关键能力 |
| 四翼 | 综合性（通项 + 求和整合） + 基础性（第 1 问考变形） |

### 4.5 考点映射

这道题挂在 **2 个考点**上：

**考点 A：递推数列求通项（构造法场景）**

| 字段 | 取值 |
|---|---|
| 主关联知识点 | 递推关系 + 等比数列通项 |
| 主关联能力 | 数学抽象 + 逻辑推理（水平 2） |
| 题型场景 | 给一阶线性递推，构造等比数列 |
| 近 5 年频次 | 8 次（高频） |
| 平均分值 | 5-6 分 |

**考点 B：错位相减法求和（n × q^n 型）**

| 字段 | 取值 |
|---|---|
| 主关联知识点 | 错位相减法 + 数列求和 |
| 主关联能力 | 数学运算（水平 3） |
| 题型场景 | bₙ = n × q^n 型求和 |
| 近 5 年频次 | 6 次 |
| 平均分值 | 7-8 分 |

### 4.6 权重信号更新

这道题的标注，会触发以下统计更新：

- 考点 A：频次 +1，分值 +5
- 考点 B：频次 +1，分值 +8
- "数学运算"素养：累计分值 +13
- "解答题中难档"槽位：累计题数 +1
- "数列模块"：累计分值 +13

跑完 5 年所有真题后，每个考点都形成稳定的**权重画像**。

### 4.7 错题归因预案（前置关联 Q5）

假设学生做错，系统能提取的根因信号：

| 出错位置 | 可能根因 | 系统下一步动作 |
|---|---|---|
| 第 (1) 没思路 | 构造法没掌握 | 推"构造法专题"3-5 题 |
| 第 (1) 思路对算错 | 计算粗心 | 仅计数，不进错题本 |
| 第 (2) 错位相减符号错 | 操作不熟 | 推错位相减专项 5 题 |
| 第 (2) bₙ 化简错 | 代数运算薄弱 | 触发前置知识点诊断 |
| 整体没动笔 | 时间不够 / 心理放弃 | 触发学习习惯干预 |

**关键约束**：要做到细粒度归因，必须让学生**分步作答 + 上传过程**（拍照 OCR 或步骤选择）。Goal Template 的考点颗粒度只决定诊断粒度上限，实际粒度还受学生作答数据粒度限制。

### 4.8 一道题在系统里的总关联数（用以建立工作量直觉）

| 关联对象 | 数量 |
|---|---|
| 考点 | 2 |
| 知识点 | 5-6 |
| 核心素养 | 3 |
| 四层四翼标签 | 2-3 |
| 卷子结构槽位 | 1 |
| 触发的统计字段 | ~10 |

**结论**：500-1000 道真题的标注是一项重投入工程，AI 能出 70% 草稿，剩下 30% 必须人工审核。

## 5. 其他必须注意的事项

### 5.1 MVP 范围的精确化

| 维度 | 取值 |
|---|---|
| 模板 | 新课标 I 卷数学 2027 v1.0 |
| 适用省份 | 安徽、山东、广东、江苏、河北、湖北、湖南、福建、浙江、辽宁等 |
| 学科 | 仅数学 |
| 题型覆盖 | 选择 / 多选 / 填空 / 解答（全覆盖） |
| 教材版本 | 仅人教版（其他版本后续扩展） |

**扩科警示**：扩到再选科目（物理、化学等）时，因为这些学科**各省自主命题**，工作量是数学的 N 倍（N = 覆盖省份数）。优先级取决于目标市场省份分布。

### 5.2 已知风险（必须写进项目风险登记册）

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 无专职教研团队 | 模板权威性不足，家长可能质疑 | MVP 话术弱化"权威"，定位"个性化辅助" |
| AI 标注错误率 | 推荐 / 诊断系统性偏差 | 强制人审，建立标注准确率监控 |
| 考纲版本更新 | 老用户数据需迁移 | MVP 必须预留模板版本号字段 |
| 真题样本不足 | 权重统计显著性不够 | 用八省联考 + 省质检扩大样本 |
| 评分细则获取困难 | 错题归因停留在粗粒度 | 教研老师手工补全（2-3 人天 / 学科） |
| 教辅版权 | 借鉴教辅考点目录可能侵权 | 仅对照不复制，保持自有体系 |

### 5.3 版本迁移策略（数据模型 MVP 必须预留）

虽然完整迁移功能可以后做，但**数据模型必须从一开始就支持**：

```
Goal Instance 表
  - id, 学生 ID
  - 模板 ID
  - 模板版本号  ← MVP 必须有
  - 起点（诊断结果快照）
  - 截止时间
  - 每日预算
  - 创建时间
  - 上次模板版本同步时间
```

迁移时需处理的三种情形：
- 知识点 / 考点在新版**保留**：直接平移
- **合并**：原 A、B 两点掌握度如何折算到新合并点 C
- **删除 / 拆分**：原数据归档策略

### 5.4 Goal Template 与 Goal Instance 的关系

```
Goal Template (运营资产)              Goal Instance (学生数据)
─────────────────                  ─────────────────
- 考试要求底座                    - 绑定的模板 ID + 版本号
- 全网学生共享                    - 学生个人的起点 / 截止 / 预算
- 由运营 + 教研维护               - 学生掌握度向量
- 有版本号                        - 学生练习历史
- 包含完整考点 / 知识点 / 权重    - 学生错题本
                                  - 一对一映射到一个学生 + 一个模板
```

**学生只能 active 一个一层目标实例**。短期专项（二层目标）通过平行赛道处理（详见后续 Q3-Q6 讨论）。

### 5.5 "考点"的产品话术

因为考点没有官方定义：

- 对家长 / 学校 ❌ 不说："对标官方考点"
- 对家长 / 学校 ✅ 应说："对标教育部课程标准 + 高考评价体系 + 历年真题"
- 对教研 ✅ 内部话术："基于真题反推 + 教辅参考的考点体系"

### 5.6 与 AI 推荐引擎的接口

Goal Template 作为推荐引擎的输入之一，需提供：

- 考点权重向量（用于采样）
- 知识点 DAG（用于前置溯源 + 学习路径生成）
- 难度分布（用于难度匹配）
- 题型结构（用于模拟卷生成）

推荐引擎的具体决策框架（**配额 → 主线 → 补强** 三层结构）：

- **第一层 必做配额**：艾宾浩斯到期项必须当天清
- **第二层 主线驱动**：按学生阶段决定（新学期跟学校进度，复习期跟考纲权重）
- **第三层 补强**：剩余时间用错题重做或弱点补强

### 5.7 阶段判定（影响主线驱动）

**阶段是 (学生 × 学科 × 目标实例) 维度独立的字段**，不是全局开关：
- 对用户展示枚举（新学期 / 巩固期 / 一轮复习 / 二轮复习 / 冲刺）
- 内部可存连续值（如"距考试紧迫度" 0-1）
- 二者互转
