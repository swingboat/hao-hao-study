# Q3 设计文档：学习闭环与"会了"的判定

> 本文档沉淀关于"如何判断学生学会了"以及"一次学习的会话怎么走"的产品决议。
>
> 关联文档：`goal_template_design.md`、`entity_relationship_design.md`
>
> 文档版本：v0.1（讨论稿）

---

## 1. 核心命题

"判断学生是否真的学会"是整个产品最难也最值钱的一环：
- 判准了 → 推荐 / 诊断 / 错题归因 / 艾宾浩斯调度全部受益
- 判不准 → 推荐系统在沙滩上跑

围绕这一命题，本轮讨论确定了 4 件事：
1. **判定的最小单位**（Q3.1）
2. **掌握度的数据形态**（Q3.2）
3. **费曼讲述的产品化形式**（Q3.3）
4. **一次学习 session 的结构**（Q3.4，同时回答了 Q4）

---

## 2. Q3.1：判定单位与展示层级

### 2.1 决议

- **对学生展示**：会了 / 没会的标签挂在**知识点（KP）层级**
- **内部维护**：同时维护**知识点 + 考点（EXAM_POINT）**两层掌握度
- **取舍理由**：考点粒度更细但对学生认知负担过重；知识点对学生友好，考点对推荐引擎友好，分开建模即可

### 2.2 落地到数据模型

- 主表：`MASTERY_STATE`（按知识点维度，必建）
- 可选聚合表：`EXAM_POINT_MASTERY`（按考点维度，MVP 可运行时计算，性能瓶颈时再物化）

## 3. Q3.2：掌握度（Mastery）

### 3.1 定义

> **Mastery（掌握度）** = 某个学生，对某个知识点（或考点），当前"会到什么程度"的量化表达。

记录在 `MASTERY_STATE.mastery_score` 字段，取值范围 0-1。

### 3.2 决议

- **内部存连续值**（0-1 float）
- **对外展示 4 档枚举**：未开始 / 学习中 / 熟悉 / 已掌握
- **二者通过阈值转换**

### 3.3 内外映射

| mastery_score（内部） | mastery_level（对外） | 含义 |
|---|---|---|
| 0.0 - 0.2 | 未开始 | 还没接触或全错 |
| 0.2 - 0.5 | 学习中 | 偶尔做对，不稳定 |
| 0.5 - 0.85 | 熟悉 | 大部分做对，有薄弱环节 |
| 0.85 - 1.0 | 已掌握 | 稳定做对，可进入长间隔复习 |

阈值非定死，MVP 上线后根据真实数据调整。

### 3.4 演化规则（示例，MVP 用简化版）

| 事件 | mastery_score 变化 |
|---|---|
| 做对基础题 | +0.05 |
| 做对中等题 | +0.10 |
| 做对难题 | +0.15 |
| 做错基础题 | -0.15 |
| 做错中等题 | -0.08 |
| 做错难题 | -0.03（难题做错不重罚） |
| 粗心错误 | 不扣分（非知识问题） |
| 讲述题达标 | +0.15 |
| N 天不练 | 按艾宾浩斯曲线衰减 |

### 3.5 Mastery 驱动的行为（产品价值）

| Mastery 区间 | 推荐引擎行为 |
|---|---|
| 接近 0 | 推学习材料（不出题），或先补前置知识点 |
| 0.2-0.5 | 推中等难度的相关题，做巩固训练 |
| 0.5-0.85 | 推变种题、跨知识点综合题，**适合插入讲述题** |
| ≥ 0.85 | 进入艾宾浩斯长间隔，**不再主动推主线题** |
| 下游 KP 的前置未达 0.3 | 阻塞推荐（前置不稳定） |

### 3.6 Mastery 不是什么（避免误解）

- 不是学生成绩 / 卷面分
- 不是跨学生排名
- 不是诊断结论
- 不是一次决定，是持续演化的状态
- 学生**永远看不到 0.65 这种数字**，只看到 4 档枚举

## 4. Q3.4：一次学习 session 的结构（同时回答 Q4）

### 4.1 决议：批量提交 + 批量分析模式

完整流程：

```
┌─────────────────────────────────────────────────────┐
│  阶段 A：答题阶段（25 分钟硬截止）                    │
│  - 系统出题填满 25 分钟（题量自适应）                  │
│  - 学生连续作答，无 LLM 介入                          │
│  - 不允许跳过（"像一份试卷一样"）                      │
│  - 可提前提交（不强制等满 25 分钟）                    │
└─────────────────────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│  阶段 B：批量 AI 分析（30-60 秒等待）                 │
│  - 一次 LLM 大调用，分析整批答题                       │
└─────────────────────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│  阶段 C：解析与建议阶段（学生自由浏览）                │
│  - 每道题：对错 + 标准答案 + 解析 + 错因诊断           │
│  - 整体：薄弱点小结 + 建议下一步                       │
│  - 每道题可点"训练相关知识点"触发 5 题次级 session     │
└─────────────────────────────────────────────────────┘
```

### 4.2 关键决策细节

| 决策点 | 取值 | 理由 |
|---|---|---|
| Session 时长 | 25 分钟硬截止 | 番茄钟 |
| 题量 | 自适应填满 | 灵活 |
| 中途卡住 | 不允许跳过，硬做 | 像试卷，避免逃避 |
| 提前完成 | 直接进入解析，不强制凑满 | 尊重学生节奏 |
| 提交方式 | 一次性整批提交 | 简化体验、降本 |
| AI 分析时延 | 30-60 秒可接受 | 集中调用代价可控 |
| "训练相关知识点" | 5 题，当场出，1 层不嵌套 | 简单优先 |
| 次级 session 数据 | 计入主线统计，打 source 标签 | 不损失信号 |

### 4.3 批量分析的输出内容

LLM 一次调用需输出：

1. 每道题：对错判定 + 标准答案 + 解析 + 错因诊断（落到 ERROR_CAUSE 枚举）
2. 本次 session 触及的知识点掌握度更新
3. 本次 session 暴露的薄弱点（聚合层）
4. 给学生的总体建议（自然语言 1-2 句）
5. 推荐的下一步动作

### 4.4 "训练相关知识点"次级 session

| 字段 | 取值 |
|---|---|
| 题量 | 5 道（可配置） |
| 题目来源 | 当场出（不预生成） |
| 嵌套深度 | 仅 1 层，不允许在次级 session 里再点 |
| 数据归属 | 计入主线统计，`source=specialized_drill` 标签 |
| 时间归属 | 不计入主线 25 分钟，是独立的时间投入 |

### 4.5 已废弃的设计

原"答题树"递归方案被砍掉：
- 原方案：对了 → 出相关知识点题；错了 → 出根因知识点题，递归
- 废弃理由：递归导致单题耗时不可控，番茄钟容易被一道题吃光，学生体验差
- 替代方案：上述"批量提交 + 学生可选触发次级 session"模式

## 5. Q3.3：费曼讲述的产品化

### 5.1 形式重定义：从"模式"降级为"题型"

**不做独立的费曼模式**，而是把讲述做成系统题库里的一种特殊**题型**：

- 新增 `ITEM_TYPE = concept_explanation`（讲述题）
- 系统在合适时机将一道讲述题作为常规题插入到 25 分钟 session 中
- 走主线的批量分析流程，工程统一

### 5.2 触发条件

讲述题不是每次 session 都出，需满足：

- 学生在某知识点上**最近 N 次答题正确率 ≥ 80%**
- 且**距上次该知识点的讲述题 ≥ M 天**（避免反复打扰）
- 且该知识点的 `mastery_score` 在 **0.6-0.85 之间**（接近但未"过"）

**每次 session 最多 1 道讲述题**（避免影响体验）。

### 5.3 三档判定（不是"讲了就过"）

| 判定档 | 条件 | 系统反应 |
|---|---|---|
| **未达标** | 字数 < 30 字 / 与知识点无关 | 不算过，建议重讲，mastery 不降 |
| **达标** | 内容相关、覆盖关键概念但有疏漏 | 算过，mastery 维持，给补充建议 |
| **优秀** | 准确、完整、有自己的语言组织 | 算过 + mastery 加成 |

**原则**：宽进但不送人头。门槛低让学生愿意讲，但拦住纯敷衍，保护讲述题在产品里的价值信号。

### 5.4 评判依据

- 由 LLM 做评判
- rubric 来自教研老师为每个知识点写的**关键点清单**（5-10 条要点）
- 覆盖一半要点即算达标
- AI 能出 70% rubric 草稿，剩 30% 人审

### 5.5 输入形式

**MVP 用打字输入**：
- 设置最小字数提示（如"建议讲 50-150 字"）
- 后续根据用户反馈再加语音
- 语音对中学生 + 数学公式场景的成本回报比不高

### 5.6 达标后 Mastery 更新

**选项**：mastery 加 0.15-0.2 + 打 `feynman_verified` 标记

不直接跳到 mastered（1.0），原因：
- 体现讲述价值，但不彻底放手
- 学生仍可能因后续答题失败回落
- `feynman_verified` 标记可作为推荐引擎的强信号独立使用

### 5.7 学生跳过 / 拒绝的处理

- 跳过算"未尝试"，不扣分，mastery 不变
- 连续跳过 N 次 → 系统暂停对该学生触发讲述题
- 提供"以后不要再让我讲这类题"关闭开关（用户控制权）

### 5.8 数据合规（重要）

讲述内容是未成年人的原创文字，必须遵守：

- **存原文用于复盘**：需要监护人同意
- **不用于训练 AI 模型**：必须在隐私政策明确
- **保留时长 ≤ 1 学年**：过期自动归档

**列入项目风险登记册。**

## 6. 数据模型补充（基于本次决议）

### 6.1 新增 ITEM_TYPE 枚举值

`concept_explanation`（讲述题）

### 6.2 新增 ATTEMPT 字段

- `feynman_explanation`: text（讲述内容）
- `feynman_score`: enum（未达标 / 达标 / 优秀）
- `feynman_rubric_coverage`: float（关键点覆盖比例）

### 6.3 新增 MASTERY_STATE 字段

- `feynman_verified`: bool
- `feynman_verified_at`: datetime
- `last_feynman_skip_count`: int（连续跳过计数，用于暂停触发）

### 6.4 LEARNING_SESSION 新增 mode

- `mainline`：主线 25 分钟番茄
- `specialized_drill`：从主线触发的 5 题次级 session
- `specialized`：二层目标专项
- `upload_analysis`：上传作业 / 错题分析

### 6.5 ATTEMPT 新增 source 字段

- `mainline` / `specialized_drill` / `specialized` / `upload`

## 7. 与其他模块的接口

### 7.1 与推荐引擎

- Mastery 向量是推荐引擎的核心输入
- 讲述题触发条件由推荐引擎判断
- `specialized_drill` 5 题由推荐引擎当场召回 + 简单过滤

### 7.2 与艾宾浩斯调度

- mastery_score ≥ 0.85 的知识点进入艾宾浩斯长间隔
- 衰减规则与艾宾浩斯曲线耦合

### 7.3 与错题本

- 答题对错由批量分析判定
- 粗心错误不进错题本
- 错因诊断结果写入 `ERROR_CAUSE`

### 7.4 与学习阶段

- 不直接耦合（PHASE_STATE 影响主线题目分布，不影响讲述触发）

## 8. 已知风险与待解决项

| 风险 | 等级 | 缓解 |
|---|---|---|
| Mastery 演化规则的合理性 | 中 | MVP 用简化版，跑通后用 BKT/IRT 精细化 |
| 批量分析 30-60 秒等待的用户耐受度 | 中 | 加进度提示、用占位预览（"正在分析第 3 题..."） |
| 讲述题 rubric 质量 | 中 | 教研老师审核为主，AI 草稿为辅 |
| 讲述题触发频率过高引起反感 | 中 | 跳过追踪 + 关闭开关 |
| 学生写入数据的合规处理 | 高 | 监护人同意 + 不训练 + 1 学年保留 |
| AI 评判讲述题的偏差 | 中 | rubric 覆盖度作为客观指标，AI 解读作为辅助 |
| 无法跳题可能让卡壳学生焦虑 | 低-中 | 监控弃做率，必要时增加"标记困难"按钮（不跳但标记） |

## 10. 决议速查（给开发 / 教研团队对齐）

1. 展示掌握度的最小单位：**知识点**
2. 内部维护：**知识点 + 考点 两层**
3. mastery 内部：**0-1 连续值**
4. mastery 对外：**4 档枚举**
5. Session 模型：**25 分钟硬截止 + 题量自适应填满 + 一次性提交 + 批量分析**
6. 不允许跳题，提前完成可提交
7. 次级训练：每题可点"训练相关知识点" → 5 题当场出，仅 1 层
8. 费曼讲述：**作为一种题型**，不是独立模式
9. 讲述触发：mastery 在 0.6-0.85 且最近答题正确率 ≥ 80%
10. 每 session 最多 1 道讲述题
11. 讲述输入：打字（MVP）
12. 讲述判定：**三档**（未达标 / 达标 / 优秀）
13. 达标处理：mastery + 0.15-0.2 + `feynman_verified` 标记
14. 数据合规：监护人同意 + 不训练 + ≤1 学年保留
