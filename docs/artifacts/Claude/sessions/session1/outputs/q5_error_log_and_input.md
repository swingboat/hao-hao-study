# Q5 设计文档：错题本与作答输入

> 本文档沉淀关于"错题如何分类记录、变种题如何生成、学生如何作答"的产品决议。
>
> 关联文档：`goal_template_design.md`、`entity_relationship_design.md`、`q3_learning_loop_and_mastery.md`
>
> 文档版本：v0.1（讨论稿）

---

## 1. 核心命题

错题本不是"垃圾桶"，是**驱动后续推荐的核心数据源**。本轮讨论确定 3 件事：

1. **错题根因如何分类**（Q5.1 - Q5.2）
2. **学生如何作答 / 系统如何采集作答数据**（Q5.3 重定义为输入方式）
3. **错题本字段与重做策略**（Q5.4 - Q5.5）

---

## 2. 错题根因分类（ERROR_CAUSE 枚举）

### 2.1 6 类根因（MVP 固定）

| 根因 | 含义 | 是否进错题本 | 后续动作 |
|---|---|---|---|
| **conceptual** | 概念性错误（知识点本身没掌握） | 是 | 回到知识点学习材料，推前置题 |
| **methodological** | 方法性错误（知识会但不会用） | 是 | 推变种题刷量（题型场景相同，参数变化） |
| **comprehension** | 审题错误（读错条件、漏看图） | 是 | 推同题型的审题训练题 |
| **computational** | 计算 / 书写粗心 | **否** | 仅计数，不进错题本 |
| **time_pressure** | 时间压力错误（不限时就能做对） | 是（标记） | 限时模式重做 |
| **out_of_scope** | 超纲 / 超难 | 是（标记） | 暂时忽略推荐 |

### 2.2 MVP 不做的细化

- 学科特定根因（如"分类讨论遗漏""时态混淆"）—— 后续按学科扩展
- 多根因的权重排序 —— MVP 用扁平的 tag 数组

### 2.3 谁来打标签

**AI 推断主导 + 学生可选修正**：

- 在解析页，每道错题旁显示 AI 推断的根因（如"根因：方法性错误"）
- 旁边一个小按钮"我觉得是..."，弹出 6 个根因选项
- 学生不主动改就用 AI 判定
- 学生修正后，`ERROR_CAUSE.inferred_by` 字段记录为 `student_confirmed`

### 2.4 AI 推断可用的信号

AI 在批量分析时能看到：
- 题干 + 标准答案 + 学生答案（OCR 识别后的文本）
- 学生答题用时
- 该知识点的历史正确率
- 该考点的题型场景

**看不到**：思考过程、是否抄、是否分心。所以 AI 推断本质是基于"对错 + 时间 + 答案差异程度"的启发式判断，**准确率预期 70-85%**。

## 3. 作答输入方式（核心架构决议）

### 3.1 设计原则

**练习要和考试同构**。学生平时键入答案、考场要写完整推导，两件事的认知负担完全不同。如果产品脱离考试现场实际，刷得再多也不能转化为分数。

### 3.2 输入方式选型

**MVP 走"路径 B：手写拍照 + AI 多模态识别"**。

理由：
1. "和考试同构"是中学市场的命门
2. 学生反正要在纸上演算，自然延伸
3. 多模态 API（GPT-4V / Claude Vision / Mathpix）成熟度已能用于生产
4. 工程量不见得比"做数学键盘"大
5. 错题本天然存图片，复盘真实

### 3.3 各题型的输入方式

| 题型 | MVP 输入方式 | 后续优化 |
|---|---|---|
| 单选 / 多选 | 键入（点选） | - |
| 填空题 | 键入文字 | 数学小键盘 |
| 解答题 | **整题手写 + 拍照上传**（可多张） | 平板手写板 / 数学键盘可选 |
| 讲述题 | 键入长文本 | 语音输入 |

### 3.4 两阶段提交流程（关键设计）

**第一阶段：答题 + 上传（25 分钟番茄钟内）**
- 客观题：键入选项 / 填空
- 解答题：在纸上做完后拍照上传（每题可多张照片）
- 不允许跳过（"像一份试卷"）

**第二阶段：OCR 解析与学生确认（番茄钟外）**
- AI 多模态识别每张照片，输出"AI 看到的文字版本"
- 学生逐题确认 AI 识别是否准确
- 错误的可点"修正"按钮，与 AI chat 修复 OCR 误识别
- 全部确认后才进入下一阶段

**第三阶段：AI 评判与解析生成（番茄钟外）**
- AI 基于学生确认后的内容做评判
- 输出对错 + 解析 + 根因诊断 + 推荐
- 显示解析页

### 3.5 OCR 防作弊机制

学生跟 AI chat 修正 OCR 时存在作弊风险（把错的改成对的）。

防护措施：

- chat 仅允许说明"这个符号是 X 不是 Y"，**不允许直接改写最终答案**
- 提供两个修正选项：**"这是我写的"** 和 **"我重新拍一张"**，禁止"直接打字覆盖"
- 修改前后的内容存日志：`ATTEMPT.ocr_correction_log`
- 一道题修正超过 N 次（如 3 次）自动锁定，触发"疑似刷分"标记

### 3.6 完整 Session 时间预估

| 阶段 | 预估时间 | 是否计入番茄钟 |
|---|---|---|
| 答题 | 25 分钟（硬截止） | 是 |
| 上传 + OCR | 30-60 秒 | 否 |
| 学生确认 OCR | 3-5 分钟 | 否 |
| AI 评判 + 分析 | 30-60 秒 | 否 |
| 学生看解析 | 5-8 分钟 | 否 |
| **总耗时** | **35-40 分钟** | - |

**体验提示**：明确告诉学生"番茄钟到点了，接下来是复盘时间，不计入专注计时"。

### 3.7 拍照上传的工程细节

- 单题多张照片支持（一道大题可能写满半页）
- 拍照质量检测：模糊 / 反光 / 倾斜 自动提示重拍
- 图片预处理：自动旋转、对比度增强
- 上传后即生成低清缩略图供 chat 上下文使用
- 原图保留至错题攻克后归档（含学生笔迹，受合规约束）

## 4. 错题本（ERROR_LOG）字段详化

### 4.1 完整字段定义

```
ERROR_LOG
├── 标识
│   ├── id (PK)
│   ├── goal_instance_id (FK)
│   └── item_id (FK)
├── 错题状态
│   ├── first_error_at
│   ├── error_count
│   ├── status (open / resolved / archived)
│   ├── resolved_at
│   └── resolved_method (variant_done / feynman_passed / manual_marked)
├── 根因诊断
│   ├── root_cause_tags (array<enum>)
│   ├── primary_kp_id
│   ├── secondary_kp_ids (array)
│   └── cause_evolution (array<{date, cause, by}>)
├── 学习轨迹
│   ├── all_attempts (array of attempt ids)
│   ├── variant_items_recommended (array)
│   └── variant_attempts (array)
├── 学生答案快照
│   ├── first_wrong_answer (text + image refs)
│   └── latest_wrong_answer (text + image refs)
├── 内容快照（防止题目更新断链）
│   ├── item_snapshot (题目内容快照)
│   ├── solution_snapshot (解析快照)
│   └── analysis_snapshot (首次 AI 分析全文)
└── 复习调度
    ├── next_review_at
    └── review_interval_days
```

### 4.2 关键设计点

**快照思路**：题目和解析必须存快照副本。如果题目后续更新或下架，学生历史错题复盘不能断链。

**根因演变**：`cause_evolution` 记录每次错题的根因变化。例如学生第一次错是 conceptual（概念不懂），第三次错是 methodological（概念懂了但方法错），说明在进步。这是面向学生展示学习曲线的关键数据。

**关联所有 attempts**：学生在这道题及其所有变种题上的答题记录串起来，形成完整学习轨迹，方便复盘和老师查看。

### 4.3 数据规模

- 单学生 MVP 阶段累积错题：50-200 条
- 单条带图片快照：约 100-500 KB（图片为主）
- 单学生错题本总量：10-100 MB
- 存储无压力，但需要 CDN 加速图片访问

## 5. 变种题与重做策略

### 5.1 按根因分流的重做策略

| 根因 | 重做内容 |
|---|---|
| conceptual | 推知识点学习材料 + 基础题（不是变种题） |
| methodological | 推变种题（题型场景相同，参数 / 表述变化） |
| comprehension | 推同题型的审题训练题 |
| computational | 不进错题本，无后续 |
| time_pressure | 限时模式重做原题 |
| out_of_scope | 暂不重做 |

### 5.2 艾宾浩斯调度（错题维度）

错题进错题本后按艾宾浩斯曲线安排重做提醒：

| 间隔 | 复习时机 |
|---|---|
| 1 天 | 首次重做 |
| 3 天 | 第二次重做 |
| 7 天 | 第三次重做 |
| 15 天 | 第四次重做 |
| 30 天 | 第五次重做（判定 resolved） |

**重做的不是原题**，是按根因策略生成的衍生题。

### 5.3 错题攻克判定（resolved 条件）

满足任一即可标记为 resolved：
- 在该错题 + 衍生题上**连续 N 次（默认 3）做对**
- 学生在对应知识点的**讲述题达标**
- 学生**手动标记**"我已掌握"（需要确认弹窗，防止误点）

## 6. 数据模型补充

### 6.1 ATTEMPT 新增字段

```
ATTEMPT
├── 原有字段...
├── answer_steps：jsonb
│   - 改为支持三层：
│     - image_urls: array<string>  // 拍照上传的原图
│     - ai_parsed_text: text       // AI OCR 识别结果
│     - student_confirmed_text: text  // 学生确认后的版本
├── ocr_correction_log：array<{from, to, by, at}>  // OCR 修正历史
├── ocr_correction_count：int                       // 修正次数
└── is_locked_for_review：bool                      // 是否因频繁修正被锁定
```

### 6.2 ERROR_CAUSE 字段补充

```
ERROR_CAUSE
├── 原有字段...
├── inferred_by：enum
│   - ai_inferred / student_confirmed / student_overridden / teacher
├── confidence：float  // AI 推断置信度
├── evidence：text     // 判断依据（如"用时仅 30 秒，疑似计算粗心"）
└── target_kp_id：FK   // 归因到的具体知识点
```

## 7. 输入方式分阶段路线图

| 阶段 | 时间 | 输入方式 | AI 评判能力 |
|---|---|---|---|
| **MVP（v1）** | 0-6 月 | 客观题键入 + 解答题拍照 | 整题对错 + 大致诊断 + 关键步骤判断 |
| **v1.5** | 6-9 月 | 加入"AI 标注作答中的关键步骤错误"的高亮回显 | 步骤级错因诊断 |
| **v2** | 9-12 月 | 加专用数学小键盘（短答案场景） | - |
| **v2.5** | 12-18 月 | 加平板手写板支持（Apple Pencil 等） | - |
| **v3+** | 18 月+ | 加语音讲解辅助 / 板书录制 | 多模态融合诊断 |

每阶段都向"和考试同构"逼近，但 MVP 不堆功能。

## 8. 已知风险

| 风险 | 等级 | 缓解 |
|---|---|---|
| 多模态 OCR 准确率 70-85% | 高 | 两阶段提交（学生确认环节） |
| 学生通过 chat 修正 OCR 时作弊 | 中 | 仅修 OCR、不准改答案、超 N 次锁定 |
| 多模态 API 成本高 | 中 | 监控单次 token 消耗、设置 session 上限 |
| 学生笔迹图片合规 | 高 | 监护人同意 + 不训练 + 攻克后归档 |
| Session 总时长 35-40 分钟超出番茄钟预期 | 低 | UI 明确告知"复盘不计入专注计时" |
| 拍照体验在低端手机上差 | 中 | MVP 提示拍照质量、后续优化预处理 |
| 错题图片存储成本 | 低 | CDN + 攻克后压缩归档 |

## 9. 决议速查

1. 6 类根因枚举，MVP 不细化
2. AI 推断 + 学生可修正
3. **解答题用手写拍照上传**，不键入
4. **两阶段提交**：上传 → OCR 确认 → 评判
5. OCR 修正 chat 仅修识别错误，不准改答案
6. 错题本详细记录（含快照 + 根因演变 + 所有 attempts）
7. 重做按根因分流（不是统一推变种）
8. 艾宾浩斯调度错题（1/3/7/15/30 天）
9. 攻克判定：连续 3 次做对 / 讲述达标 / 学生手动标记
10. 数学键盘 / 手写板等列入路线图，不进 MVP
11. 单次 session 总耗时 35-40 分钟，复盘环节不计入番茄钟
12. 学生笔迹图片受未成年人合规约束（监护人同意 + 不训练 + 攻克后归档）
