# Q1、Q3、Q5 决议总结

## Q1 学习目标 / Goal Template 决议总结

### 核心问题

如何建立"课程标准 + 高考要求 + 知识点 DAG + 考点体系"的四元组模板系统，作为推荐引擎的上位法和学生诊断的权威依据。

### 关键决议

#### 1. Goal Template vs Goal Instance 的分离

- **Goal Template**（运营资产）：课程标准版本、高考卷种、学科、省份的组合，包含课标锚点、知识点 DAG、能力体系、考点权重、卷子结构、真题库
- **Goal Instance**（学生数据）：学生个人创建的学习目标，必须绑定一个模板版本号、截止时间、起点成绩
- **关键字段**：Goal Instance 必须在 MVP 就预留 `template_version` 字段，为未来的考纲迁移策略预留通道

#### 2. 完整的 Goal Template 包含 4 类数据源

| 数据源 | 来源 | 内容 | 版本管理 |
|---|---|---|---|
| **课程标准** | 教育部官网免费下载 | 《普通高中数学课程标准 2017 年版 2020 年修订》| 全国统一一版 |
| **高考评价体系** | 教育部考试中心 + 人教社出版书 | 一核四层四翼；核心素养水平 1-3 | 全国统一 |
| **考试实施方案** | 安徽省教育招生考试院官网 | 3+1+2 模式、分值结构、题型比例、时长、赋分规则 | 按年份更新 |
| **历年真题** | 教育部考试中心 + 省级质检 + 八省联考 | 近 5-10 年真题 + 官方解析 + 难度系数 | 滚动收集 |

#### 3. 完整的 Goal Template 必须补充的资料

**必须补充（MVP 不能漏）**：
- 《课程标准解读》（定义粒度）
- 《高考评价体系说明》（四层四翼怎么用）
- 《高考试题分析》数学分册（近 5 年，官方命题解读）
- 人教版教师用书全套（重难点 + 前置关系）
- 近 5 年新课标 I 卷真题 + 官方答案与评分细则
- 近 3 年八省联考 + 安徽省质检卷
- 安徽省最新考试实施方案（官网）
- 1-2 本主流教辅（《五三》《一本》）作为考点参考

**资料成本**：1000 元以内；**工作量**：30-45 人天（1 个高中数学教研老师 + AI 辅助，3-4 个月）

#### 4. 知识点 DAG 与考点体系的定义

- **知识点（KP）**：学生在一个单一概念 / 技能上的学习单元，学习时间 15-30 分钟
- **知识点 DAG**：知识点之间的"前置关系"有向边（不是树而是图）
- **考点 = KP × 能力要求 × 题型场景**：一个知识点可衍生 3-8 个考点；一个考点可涉及多个知识点（多对多）
- **能力维度**：直接用课标的 6 大核心素养 + 高考评价体系的四层四翼
- **权重**：从历年真题统计每个考点的出现频次 + 平均分值，作为推荐采样依据
- **规模**：高中数学 200-400 个考点是合理量级

#### 5. Q1 明确不做或留待后续的事项

**MVP 不做**：
- 其他学科的课纲模板（仅做数学）
- 其他卷种的模板（仅做新课标 I 卷）
- 教研团队建设（目前不招，先用自审 + 兼职老师）
- 考纲版本迁移功能（字段预留，功能 v1.5+）

**已知风险标记**：
- 教研专家缺失会影响模板权威性，融资 / 扩科时要回头算账
- MVP 期间产品话术"不打权威"，定位"个性化练习辅助"

## Q3 学习闭环与"会了"的判定决议总结

### 核心问题

如何定义和演化"学生掌握度"这个关键状态，驱动推荐引擎、阶段判定、艾宾浩斯调度。

### Q3.1 判定单位决议（知识点层级）

- **对学生展示**："会了 / 没会"标签挂在知识点层级，不让学生看考点细节
- **内部同时维护**：知识点和考点两层掌握度，推荐引擎用考点级做精准采样

### Q3.2 掌握度决议

- **内部存储**：连续值 0-1（mastery_score）
- **对外展示**：4 档枚举（未开始 / 学习中 / 熟悉 / 已掌握）
- **阈值**：0.2 / 0.5 / 0.85

### Q3.3 费曼讲述决议（降级为"一种题型"）

- **形式**：不做独立模式，而是讲述题（item_type=concept_explanation）穿插日常
- **触发条件**：
  - 最近 N 次正确率 ≥ 80%
  - 距上次讲述题 ≥ M 天
  - mastery_score 在 0.6-0.85 之间
- **输入形式**：打字，50-150 字建议
- **评判三档**：未达标 / 达标 / 优秀
- **mastery 更新**：讲述达标 +0.15-0.2

### Q3.4 番茄钟与批量分析决议

- **模式**：固定 25 分钟硬截止，题量自适应填满
- **提交**：一次性提交进入批量分析（不递归）
- **分析内容**：对错 + 根因诊断 + mastery 更新 + 下一步建议
- **二级触发**：解析页可点"训练相关知识点"出 5 道相关题（预生成模式秒出）

### Q3 收口决议

- 内部连续外部枚举分离
- 讲述题融入主线
- 费曼三档评判防虚设
- 25 分钟硬截止
- 批量分析链路统一

## Q5 错题本与作答输入决议总结

### 核心问题

如何通过多模态识别、两阶段提交、六类根因诊断、反向调度，把"做错"转化为"定向补强"。

### Q5.1 六类错题根因

| 代码 | 名称 | 后续动作 |
|---|---|---|
| **CONCEPTUAL** | 概念性错误 | 推知识点学习 + 前置诊断 |
| **METHODOLOGICAL** | 方法性错误 | 变种题刷量 |
| **COMPREHENSION** | 审题错误 | 审题训练题 |
| **COMPUTATIONAL** | 计算粗心 | 仅计数不进错题本 |
| **TIME_PRESSURE** | 时间压力 | 限时模式重做 |
| **OUT_OF_SCOPE** | 超纲超难 | 暂不推荐 |

### Q5.2 打标签者

- **谁打**：AI 推断为主
- **学生修正**：解析页可选修正，弹窗选 6 类
- **修正记录**：ATTEMPT.cause_correction_log 存审计

### Q5.3 作答方式（两阶段提交）

**第一阶段：拍照 + OCR 确认**
- 客观题：键入
- 解答题：拍照上传手写过程
- 系统调多模态 LLM 识别
- 学生逐题确认

**第二阶段：OCR 修正**
- 学生可点"修正"与 LLM chat
- chat 仅限修正 OCR 误识别，禁止改答案
- 修正超 N 次自动锁定（防作弊）

**第三阶段：LLM 批量评判**
- 拿到确认后的完整答案进行评判

### Q5.4 错题本字段

- 错题状态（首次错、累计错次、当前状态、攻克时间）
- 根因诊断（标签、主知识点、根因演变历史）
- 作答快照（首次错、最近错、OCR 修正日志）
- 题目和解析快照（完整备份，防断链）
- 学习轨迹（所有 attempts 关联、变种题推荐、变种作答）
- 艾宾浩斯调度（下次复习、当前间隔、复习历史）

### Q5.5 重做策略与艾宾浩斯

**按根因分流**：
- CONCEPTUAL → 学习材料 + 前置诊断（立即 + 7 天）
- METHODOLOGICAL → 变种题（1 / 3 / 7 / 15 天）
- COMPREHENSION → 审题题（1 / 3 天）
- COMPUTATIONAL → 不重做
- TIME_PRESSURE → 限时模式（3 / 7 天）
- OUT_OF_SCOPE → 暂不推荐

**艾宾浩斯规则**：
- 间隔点：1、3、7、15、30 天
- 成功时间翻倍
- 失败重置为 1 天
- 讲述达标直接跳 30 天

### 输入路线图

| 版本 | 时间 | 解答题 | 特点 |
|---|---|---|---|
| **MVP** | 0-6 月 | 拍照+多模态 | 两阶段提交闭环 |
| **v1.5** | 6-9 月 | 拍照+步骤高亮 | 步骤级诊断 |
| **v2** | 9-12 月 | 拍照+数学键盘 | 短答案优化 |
| **v2.5** | 12-18 月 | 手写板支持 | 多模态融合 |

### Q5 决议速查

- 六类根因：CONCEPTUAL / METHODOLOGICAL / COMPREHENSION / COMPUTATIONAL / TIME_PRESSURE / OUT_OF_SCOPE
- 打标签：AI 推断，学生可选修正
- 作答：拍照+多模态，客观题键入
- 两阶段：OCR 确认 → 提交评判
- 防作弊：修正仅限 OCR 误识别，禁改答案，超 N 次锁定
- 艾宾浩斯：1-3-7-15-30 天，成功翻倍，失败重置

Q2（学习阶段判定）被用户主动跳过，未独立展开。但从 Q1、Q3 已隐含阶段判定原理：

- 阶段由学生填入或系统按时间推导
- 阶段判定按学科独立
- 推荐引擎根据阶段决定选题策略（新学 vs 复习 vs 冲刺）
- 三层决策框架（艾宾浩斯到期项 → 主线阶段驱动 → 弱点补强）已认可
