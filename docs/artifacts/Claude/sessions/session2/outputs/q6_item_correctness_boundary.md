# Q6 题目正确性边界

> 决定学生看到的每道题从哪里来、如何保证不出错。这是教育产品权威性的命门。

## 1. 背景

数学题的"正确性"是绝对的——答案错了就是错了，没有解释空间。学生练了答案错的题，对产品信任会直接崩塌。家长发现答案错误会迅速传播。本文档锁定 MVP 阶段的题目来源、校验、举报、下架的完整链路。

## 2. 题目来源策略（Q6.1）

采用 **B 为主 + C 补充 + 不做 A** 的混合方案。

| 方案 | 是否采用 | 说明 |
|---|---|---|
| A 纯 LLM 生成 | 否 | 正确率太低（30-60%），不在 MVP |
| B 题库召回 | **主力** | 真题 / 联考 / 质检卷，500-1000 道 MVP 起步 |
| C 题库召回 + LLM 改写 | 补充 | 仅做参数级改写，缓解题库重复率 |

## 3. LLM 改写边界（Q6.2）

| 改写类型 | 允许 | 说明 |
|---|---|---|
| 改数字 / 参数（a₁=1 → a₁=2） | 允许 | 必须 AI 自验答案 |
| 改题目情境包装（数学题加实际场景） | 允许 | 改写后需校验语义保真 |
| 改题型（选择题改解答题） | **禁止** | 风险高 |
| 改考点 | **禁止** | 风险高 |

## 4. 入库校验链路（Q6.3）

**MVP 阶段教研团队未到位，采用 AI 自动过审 + 强护栏。教研团队到位后切换至人工终审。**

### 4.1 分级自审策略

| 题源类型 | 自审策略 | MVP 入池 |
|---|---|---|
| 客观题 + 题库直接召回 | 直接入池 | 是 |
| 客观题 + LLM 参数改写 | 双 LLM 交叉验答 + 答案一致才入池 | 是 |
| 解答题 + 题库直接召回 | 直接入池 | 是 |
| 解答题 + LLM 参数改写 | 双 LLM 交叉验答 + 解题步骤合理性 AI 评分 ≥ 阈值 | 是 |
| 任何纯 LLM 生成题 | 禁止入池 | 否 |

### 4.2 灰度池机制

LLM 改写产生的新题 → 先进灰度池：

- 仅推给 5-10% 学生
- 灰度期 7 天
- 期间举报率 < 1% 且 ≥ 20 个学生做过 → 进主推荐池
- 期间被举报 ≥ 2 次 → 自动下架进复审队列

## 5. 学生举报机制（Q6.4）

### 5.1 举报入口

解析页每道题旁边一个"这题有问题"按钮。

### 5.2 举报选项

- 题干歧义
- 答案错误
- 解析错误
- 超纲
- 其他（带文本输入）

### 5.3 自动下架阈值

| 题所在池 | 下架阈值 |
|---|---|
| 主推荐池 | 3 次独立学生举报 |
| 灰度池 | 2 次独立学生举报 |

### 5.4 下架后的事后兜底

- 所有做过该题的学生 → 该题不计入 mastery 演化（回滚）
- 错题本里该题标 `invalidated`，不计入错题统计
- 主动推送通知给学生："这道题答案有误，已从你的记录中剔除"

## 6. 题目元数据（Q6.5）

### 6.1 MVP 必填字段

| 字段 | 说明 |
|---|---|
| `kp_ids[]` | 涉及知识点（可多个，标主次） |
| `exam_point_ids[]` | 涉及考点（知识点 × 能力 × 题型场景） |
| `difficulty` | 1-5 难度等级 |
| `item_type` | 选择 / 填空 / 解答 / 讲述 |
| `scenario_tags[]` | 题型场景标签 |
| `expected_time_sec` | 预估解题用时 |
| `source` | 真题 / 联考 / 质检 / AI 改写 |
| `source_year`, `source_region` | 真题来源年份地区（可选） |

### 6.2 留到 v1.5 的字段

- `cognitive_level`（布鲁姆分类）
- `prereq_kp_ids[]`（前置知识点，影响推荐阻塞逻辑）

## 7. 风险登记

| 风险 | 等级 | 缓解 |
|---|---|---|
| AI 自审准确率 85-92%，预估 MVP 题库 8-15% 题目带错 | 高 | 灰度池 + 学生举报 + 事后回滚 |
| 没有教研终审兜底，全靠学生举报发现错误 | 高 | 教研团队招聘列为 MVP 启动前置依赖，至少 3 个月内到位 |
| 一旦舆情事件，需要 24 小时内全量复审能力 | 中 | 预留客服 + 紧急下架工具 |
| 学生恶意举报（刷举报让题下架） | 低 | 同一学生对同一题仅算 1 次举报；高频举报学生标记审查 |

## 8. 决议速查

1. 题目来源：B 题库召回为主 + C 参数改写补充，不做 A 纯生成
2. 改写边界：仅改数字 / 参数 / 情境，禁改题型 / 考点
3. MVP 校验：AI 自动过审 + 分级策略 + 灰度池 + 事后回滚
4. 教研团队到位后切换至人工终审为主
5. 双 LLM 交叉验答用于所有 LLM 改写题
6. 灰度池 5-10% 学生 / 7 天观察期
7. 主推荐池举报阈值 3 次自动下架
8. 灰度池举报阈值 2 次自动下架
9. 下架题对所有学生的 mastery / 错题本回滚
10. 主动推送告知学生答案有误（信任修复）
11. MVP 元数据只打必填项
12. cognitive_level / prereq_kp_ids 留到 v1.5
13. 同一学生对同一题仅计 1 次举报
14. 教研团队招聘列为 MVP 启动前置依赖

## 9. 与其他 Q 的关联

- **Q3 mastery 演化**：下架题需回滚 mastery，需 MASTERY_STATE 支持事务回退
- **Q5 错题本**：错题需支持 `invalidated` 状态
- **Q7 推荐引擎**（下一站）：灰度池作为独立池源参与召回
- **Q4 番茄钟**：举报操作放在解析页，不打断答题阶段
