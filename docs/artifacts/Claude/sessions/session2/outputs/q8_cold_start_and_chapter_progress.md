# Q8 冷启动策略与章节进度

> 新学生入驻第一周的体验设计 + 数据初始化。MVP 不做摸底卷，走渐进式冷启动。

## 1. 背景与核心决策

新学生入驻时系统什么都不知道（mastery 全空 / 错题本空 / 艾宾浩斯队列空），但学生第一次打开就期待立即能开始学。冷启动要在"快速可用"和"诊断准确"之间找平衡。

**核心决策：MVP 不做摸底卷。**

理由：

1. **统计上摸不准**：30 道题覆盖 30-50 个 KP，平均 1 道/KP，无法可靠估算 mastery
2. **测试环境失真**：无监督 / 选择题蒙题率 / 心理状态像考试 / 一次性集中做注意力衰减
3. **不如正式学习准**：3-5 个 session 后两条路殊途同归
4. **本质是安慰剂**：给学生仪式感 + 给运营交付物，但对推荐系统实际价值有限

**替代方案：渐进式冷启动**——所有 mastery 从 0 开始演化，靠正式答题数据 + 难度自适应自然建立画像。

## 2. 入驻信息采集（Q8.1）

### 2.1 必填项（5 个）

| 字段 | 用途 |
|---|---|
| 年级（高一 / 高二 / 高三） | 决定题库 KP 范围 |
| 目标考试（高考 / 中考 / 期中期末） | 创建 Goal Instance |
| 章节进度（章 + 节级） | 决定可推 KP 边界 |
| 每日可投入时长 | 决定 session 题量 / 一天 session 数 / 疲劳保护 / 临考进度预警 |
| 省份 + 教材版本 | 决定章节-KP 映射表 |

### 2.2 选填项

- 目标分数 / 目标学校
- 自评弱项 / 强项

### 2.3 每日时长的 4 处用途

1. **单 session 题量**：时长 ÷ 平均 expected_time → 题数
2. **一天 session 数上限**：决定艾宾浩斯排期密度
3. **疲劳保护**：连续做超过设定时长 1.5 倍触发提示
4. **临考进度预警**：剩余天数 × 日时长 < 未覆盖 KP 所需时间 → 进度风险

## 3. 章节进度（Q8 新增）

### 3.1 章节进度 vs 知识点进度

两个独立变量，语义不同：

| 维度 | 章节进度 | 知识点进度 (mastery) |
|---|---|---|
| 存储 | `STUDENT.current_chapter` 单值 | `MASTERY_STATE` 每 KP 一条 |
| 更新者 | 学生自报 / 系统软推断 | 系统据答题自动演化 |
| 更新频率 | 每周 1 次左右 | 每 session 后 |
| 数据类型 | 离散：第 X 章第 Y 节 | 连续：0-1 |
| 作用 | **边界**——哪些 KP 允许被推 | **内容**——在允许范围内学到什么程度 |
| 学生看到 | "学到第 5 章第 2 节" | "等差数列：熟悉中" |

### 3.2 关系链

```
章节进度 → 解锁哪些 KP 可学 → KP 进 new_knowledge 池 → 学生做题 → 演化 mastery
```

### 3.3 对推荐池的边界作用

| 池 | 取数边界 |
|---|---|
| `new_knowledge` | 仅"已学范围"内 |
| `error_book_variant` | 仅已学范围内积累的错题 |
| `ebbinghaus_due` | 同上 |
| `feynman` | 同上 |
| `comprehensive`（v1.5） | 同上 |

未学章节的 KP **不进任何池**，硬推会让学生懵。

### 3.4 决策矩阵

| 议题 | 决议 |
|---|---|
| 更新方式 | 每周日主动提示 + 一键确认；系统观察到学生大量正确做后续章节题时温和软提示 |
| 颗粒度 | 章 + 节级（下拉选择） |
| MVP 教材版本 | **仅人教版高中数学**；其他版本 v2 扩展 |
| 超前解锁 | 默认按学校进度，学生可主动勾选解锁未学章节进 `new_knowledge` 池 |

### 3.5 隐性问题

章节进度告诉系统"老师教过的范围"，**不代表学生掌握的范围**。后者必须靠真实答题数据演化 mastery。所以**章节进度 ≠ mastery**。

## 4. 渐进式冷启动机制

### 4.1 初始化

学生完成 5 项自报后：

- 所有可学范围 KP 的 `mastery = 0`
- 错题本空 / 艾宾浩斯队列空
- 标记 `STUDENT.cold_start_mode = true`

### 4.2 第 1 周配比（仅新知识点池）

```yaml
new_student_week_1:
  - pool: new_knowledge
    target_ratio: "1.0"
    # 难度起点：1-2 基础题为主
    # 难度自适应敏感度调高：错 1 道立即降难度 / 对 2 道立即升难度
```

**额外保护**：
- 不触发讲述题（mastery 数据不足）
- 不触发综合题（v1.5 之后才有）
- 单道题超过 `expected_time × 2` 自动提示"是否跳过"

### 4.3 第 2 周配比（错题与艾宾浩斯开始介入）

```yaml
new_student_week_2:
  - pool: new_knowledge
    target_ratio: "0.5"
  - pool: error_book_variant
    target_ratio: "0.3"
  - pool: ebbinghaus_due
    target_ratio: "0.2"
```

### 4.4 第 3 周起：正常配比

切换到 `default` 配比（参见 Q7 推荐引擎文档），并清除 `cold_start_mode` 标记。

## 5. 冷启动结束判定

**满足任一即结束**：

- 完成 ≥ 5 个正式 session
- 入驻满 7 天
- 学生手动选择"我已熟悉系统"跳过

结束后：
- 清掉 `cold_start_mode` 标记
- 切换到正常配比
- 难度自适应敏感度恢复默认
- 解锁讲述题触发

## 6. 决议速查

1. MVP 不做摸底卷
2. 入驻必填 5 项：年级 / 目标考试 / 章节进度 / 每日时长 / 省份+教材
3. 每日时长用途：单 session 题量 / 日 session 数 / 疲劳保护 / 临考预警
4. 章节进度与 mastery 是两个独立变量，前者是边界后者是内容
5. 章节进度更新：每周日提示 + 自动推断软提示
6. 章节颗粒度：章 + 节级
7. MVP 教材：仅人教版高中数学
8. 超前解锁：学生可主动勾选未学章节
9. 初始 mastery 全部 = 0，靠演化
10. 第 1 周仅 new_knowledge 池，难度自适应敏感度调高
11. 第 1 周不触发讲述题 / 综合题
12. 第 2 周配比：new 50 / error 30 / ebbinghaus 20
13. 第 3 周切正常配比
14. 冷启动结束：≥5 session 或 入驻 7 天 或 学生手动跳过
15. v2+ 可考虑加 5-10 道轻量分档卷作为运营钩子（有真实数据后再评估）

## 7. 风险登记

| 风险 | 等级 | 缓解 |
|---|---|---|
| 第 1 周无错题数据，推荐质量依赖 KP 标注精度 | 中 | 难度自适应敏感度调高 + 单题超时提示 |
| 学生不更新章节进度导致推荐越界 / 滞后 | 中 | 每周日主动提示 + 系统软推断兜底 |
| 教材版本限制初期市场覆盖 | 中 | MVP 聚焦人教版高中数学 + v2 扩展 |
| 学生超前自学解锁后表现差，挫败感 | 低 | 解锁时弹窗提示"该章节学校尚未教，建议先做基础题" |

## 8. 与其他 Q 的耦合

| 引擎组件 | 依赖 |
|---|---|
| 章节进度边界 | Q6.5 题目元数据的 `kp_ids[]` + 教材-章节-KP 映射表（教研维护） |
| 渐进式配比 | Q7.2 配比表达式引擎需支持 `new_student_week_1/2` 阶段 |
| 难度自适应 | Q7 新知识点池的难度逻辑 |
| 冷启动结束 | Q3 mastery 演化 / Q5 错题本 / Q7 艾宾浩斯队列从此正式启动 |
