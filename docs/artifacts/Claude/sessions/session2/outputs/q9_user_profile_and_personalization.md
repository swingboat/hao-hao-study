# Q9 用户画像与个性化深度

> 决定系统对学生"了解"到什么程度，以及个性化体现在哪些产品决策。MVP 保守，不过度个性化。

## 1. 背景

到目前为止系统对学生的了解主要靠 mastery 向量 + 错题本 + 章节进度，但学生还有大量学习行为偏好未被建模。Q9 决定 MVP 阶段画像做多深、激励机制怎么设计、学生 / 家长侧分别能看什么。

**核心原则**：

1. 画像影响推荐"权重"而非"边界"，避免信息茧房
2. 中学生不是小学生，幼稚激励反感；高三不是初一，临考期激励逻辑不同
3. 学生信任是教育产品命门，家长信息访问保守

## 2. 画像维度（Q9.1）

| 类别 | 字段 | MVP |
|---|---|---|
| **基础属性** | 年级 / 性别（选填）/ 省份 / 教材版本 | 做（入驻已采） |
| **学习能力** | 平均答题速度 / 整体正确率 / 难度承受 / mastery 平均涨幅 | 做（系统自动统计） |
| **行为偏好** | 抗拒题型 | 做（仅此项） |
| **行为偏好** | 偏好题型 / 偏好时段 / 偏好难度起点 | 不做（v1.5） |
| **心理 / 动机** | 连续天数 / 放弃率 / 主动挑战率 / 挫败容忍度 | 不做（数据稀疏） |

**理由**：MVP 阶段只采"零成本能算出来的"和"对推荐质量影响大的"两类信号，其他等真实数据起来再决定。

## 3. 偏好捕获与应用（Q9.2）

### 3.1 抗拒信号识别

| 信号 | 推断 |
|---|---|
| 某题型连续 3 次跳过 / 留空 / 极短时间提交错答 | 抗拒该题型 |
| 某时段答题正确率显著高 | 偏好时段（v1.5 才用） |
| 错 1 题后连续错 3+ 题 | 挫败传染 |

### 3.2 应用方式：弱应用

| 信号 | 应用 |
|---|---|
| 抗拒题型 | **降权但不剔除**（出现频率 ×0.5） |
| 挫败传染 | 错题后插一道简单题恢复信心 |

**禁止做**：

- 完全不推抗拒题型（高考不管你喜不喜欢）
- 完全按偏好难度推（学不到上限）
- 心情差自动减题量（不可靠且会被薅羊毛）

## 4. 激励机制（Q9.3）

### 4.1 MVP 做的

| 机制 | 设计 |
|---|---|
| 连续学习天数（streak） | 显示数字；提供"补救卡"（错 1 天可保护，每周 1 张） |
| mastery 进步周报 | 学生自己看，不强推家长 |
| 错题攻克庆祝 | 简单动效，不夸张，频次受控 |
| 每日推荐量提示 | "今天 12 道题等你"——朴素有效 |

### 4.2 MVP 不做

| 机制 | 不做理由 |
|---|---|
| 挑战榜 / 排名 | 风险大于收益（中后段学生压力 / 家长拿来管教损伤使用意愿） |
| 虚拟奖励 / 徽章 | 中学生不买账 |
| 家长可见的进步报告 | v1.5 单独做家长端再说 |

### 4.3 阶段化激励逻辑

- 正常期：进步可视化 + streak + 错题攻克
- 临考期（≤ 30 天）：**切换为"焦虑感"激励**——突出"距离高考剩 X 天 / 还有 Y 个 KP 未掌握"
- 冲刺期（≤ 7 天）：仅保留"今日复习清单"，去掉所有娱乐化元素

## 5. 学生控制项（Q9.4）

**核心原则**：给"调节奏"权，不给"破坏学习闭环"权。

### 5.1 MVP 提供

| 设置项 | 说明 |
|---|---|
| 每日学习时长 | 必做 |
| 提醒时间 | 必做 |
| 跳过讲述题 | 讲述题前 3 次跳过即自动暂停触发 |
| 关闭挑战题弹窗 | 学有余力学生可关 |
| 调整难度起点 | "太难了"按钮，临时降一档 |

### 5.2 MVP 不提供

| 设置项 | 不做理由 |
|---|---|
| 调整推荐配比 | 普通用户搞不懂 |
| 关闭某 KP 的推荐 | 违反目标系统逻辑 |
| 切换学科 | v1.5（MVP 只做数学） |

### 5.3 兴趣化解析（讨论后 MVP 不做）

讨论了"让 AI 解析时按学生兴趣加趣味元素（萌系 / 运动 / 科幻等风格）"。

**决议**：**MVP 不做**，留 v1.5 再评估。

风险记录（v1.5 评估时参考）：

- IP 侵权（不可用具体商标名）
- AI 生成质量参差，弱比喻显幼稚
- 高年级反感率高
- LLM 成本增加 10-20%
- 高三 / 临考期需强制关闭

## 6. 家长侧画像（Q9.5）

**MVP 家长端不做**。

**理由**：

- 学生信任是教育产品命门，家长监控会损伤使用意愿
- 家长端是独立产品形态，工程量大
- MVP 聚焦学生学习闭环

**后续路径**：v1.5 单独立项做家长端，需先解决：

- 学生侧是否同意家长可见（设置项）
- 家长可见数据的脱敏（积极语言摘要而非原始数据）
- 是否做家长付费 / 学生使用的分离支付模型

## 7. 决议速查

1. MVP 画像范围：基础属性 + 学习能力 + 抗拒题型
2. 不做心理动机维度
3. 偏好弱应用：降权不剔除
4. 错题后插简单题恢复信心
5. 激励：streak + 进步周报 + 错题庆祝 + 每日量提示
6. 不做挑战榜 / 虚拟奖励 / 家长报告
7. 临考期切换为焦虑感激励
8. 冲刺期去娱乐化
9. 学生可控：时长 / 提醒 / 跳讲述 / 关挑战弹窗 / 临时降难度
10. 学生不可控：配比 / 关 KP / 换学科
11. 兴趣化解析 MVP 不做
12. 家长端 MVP 不做

## 8. 风险登记

| 风险 | 等级 | 缓解 |
|---|---|---|
| 信息茧房（偏好强应用导致学生回避） | 高 | 弱应用 + 定期"突破舒适区"推荐 |
| streak 中断打击留存 | 中 | 补救卡机制 |
| 进步周报反向打击（进步慢时） | 中 | 用积极语言措辞，强调长期累积 |
| 家长缺席 → 付费意愿弱 | 中 | v1.5 补家长端，MVP 验证学生留存先 |
